# Jolt 刚体物理核心算法 — 数据结构定义

## 一、刚体 (Body) 的最小必要字段

从 Jolt 的 `Body.h` 和 `MotionProperties.h` 中提取，仿写只需要以下字段：

```c
struct RigidBody {
    // ======== 几何状态 ========
    Vec3   position;          // 质心世界位置 (m)
    Quat   rotation;          // 朝向四元数 (单位四元数)
    Shape* shape;             // 碰撞形状引用 (球/盒/凸体)

    // ======== 运动状态 ========
    Vec3   linearVelocity;    // 质心线速度 (m/s)
    Vec3   angularVelocity;   // 角速度 (rad/s)，世界空间

    // ======== 质量属性 (仅 Dynamic 需要) ========
    float  invMass;           // 逆质量 = 1/m (1/kg)
    Vec3   invInertiaDiag;    // 逆惯性张量对角线 D = (dx, dy, dz)
    Quat   inertiaRotation;   // 惯性主轴旋转 R
    // 关系: I_body^{-1} = R * diag(D) * R^{-1}

    // ======== 力/扭矩累积 (每帧清零) ========
    Vec3   force;             // 累积力 (N)
    Vec3   torque;            // 累积扭矩 (N·m)

    // ======== 材质属性 ========
    float  friction;          // 摩擦系数 (无量纲, 通常 0~1)
    float  restitution;       // 恢复系数 (0=完全非弹性, 1=完全弹性)

    // ======== 阻尼 ========
    float  linearDamping;     // 线性阻尼系数 c (v *= max(0, 1-c*dt))
    float  angularDamping;    // 角阻尼系数
    float  gravityFactor;     // 重力缩放因子 (1.0=正常, 0=无重力)

    // ======== 类型 ========
    enum MotionType {
        Static,     // 不可移动 (invMass=0, 无MotionProperties)
        Kinematic,  // 按速度移动，但不受力影响 (invMass视为0)
        Dynamic     // 受力+碰撞完全影响
    } motionType;
};
```

## 二、惯性张量的对角化存储

Jolt **不存储完整 3×3 逆惯性矩阵**。在创建刚体时，通过
`MassProperties::DecomposePrincipalMomentsOfInertia()` 做特征值分解：

```
I_body^{-1} = R · diag(dx, dy, dz) · R^{-1}
```

存储为:
- `invInertiaDiag` = (dx, dy, dz)  — 3个float
- `inertiaRotation` = R            — 1个Quat (4个float)

**对比完整矩阵存储 (9个float)**，节省内存且计算更快。

### 世界空间逆惯性乘向量

这是最频繁的操作，Jolt 在 `MotionProperties.inl` 中实现：

```
MultiplyWorldSpaceInverseInertiaByVector(bodyRotation, v):
    // 组合旋转: body朝向 * 惯性主轴旋转
    combinedRot = bodyRotation * inertiaRotation
    rotMatrix = Mat44::sRotation(combinedRot)

    // 转到惯性主轴空间 → 逐分量乘对角线 → 转回世界空间
    localV = rotMatrix^T * v          // 世界→本体主轴
    scaled = invInertiaDiag ⊙ localV  // 逐分量乘
    result = rotMatrix * scaled       // 本体主轴→世界
    return result
```

### 常见形状的惯性张量

| 形状 | 惯性张量 I (质量m) | 说明 |
|------|-------------------|------|
| 实心球 (半径r) | (2/5)mr² · I₃ | 对角线全相等 |
| 实心长方体 (a×b×c) | m/12 · diag(b²+c², a²+c², a²+b²) | 对角化已完成 |
| 实心圆柱 (半径r, 高h) | diag(m(3r²+h²)/12, m(3r²+h²)/12, mr²/2) | 绕轴对称 |

对角化后 `inertiaRotation` 对于上述对称形状就是单位四元数。
只有不对称凸体才会有非平凡的旋转。

## 三、接触点数据结构

```c
struct ContactPoint {
    Vec3  position1;   // body1 表面上的接触点 (质心相对坐标)
    Vec3  position2;   // body2 表面上的接触点 (质心相对坐标)
    float penetration; // 穿透深度 (>0 表示穿透, <0 表示分离)
};

struct ContactManifold {
    Body* body1;
    Body* body2;
    Vec3  normal;       // 碰撞法线 (世界空间, body1→body2 方向)
    Vec3  tangent1;     // 切线方向1 (与 normal 正交)
    Vec3  tangent2;     // 切线方向2 (与 normal、tangent1 正交)
    float combinedFriction;
    float combinedRestitution;
    ContactPoint points[4];  // 最多4个接触点
    int   numPoints;
};
```

Jolt 的做法：摩擦和恢复系数默认取两体的**几何平均值**:
- `combinedFriction = sqrt(body1.friction * body2.friction)`
- `combinedRestitution = max(body1.restitution, body2.restitution)`

这些可以在 `PhysicsSystem::SetCombineFriction/Restitution` 中自定义。

## 四、碰撞形状的 Support 函数接口

GJK/EPA 不关心具体形状，只需要一个 `GetSupport(direction)` 函数：

```c
// 返回形状在 direction 方向上最远的点
Vec3 GetSupport(Vec3 direction);
```

常见形状的 Support 函数:

| 形状 | GetSupport(d) |
|------|--------------|
| 球 (中心c, 半径r) | `c + r * normalize(d)` |
| 长方体 (半尺寸h) | `(sign(d.x)*h.x, sign(d.y)*h.y, sign(d.z)*h.z)` |
| 凸多面体 (顶点集V) | `argmax_{v∈V} (v · d)` |
| 胶囊体 | 两球心取更远的 + r * normalize(d) |

Jolt 在 `ConvexSupport.h` 中还提供了组合包装:
- **`MinkowskiDifference<A,B>`**: `A.GetSupport(d) - B.GetSupport(-d)`
- **`AddConvexRadius<A>`**: `A.GetSupport(d) + radius * normalize(d)`
  （在 GJK 核心中不含 convex radius，EPA 中才包含）

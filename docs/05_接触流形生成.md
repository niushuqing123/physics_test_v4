# Jolt 刚体物理核心算法 — 接触流形生成

对应 Jolt 源码: `Geometry/ClipPoly.h` 和
`Physics/Collision/ManifoldBetweenTwoFaces.cpp`。

## 一、为什么需要接触流形？

GJK/EPA 只返回**一个接触点对**。但物理模拟需要**多个接触点**:
- 一个盒子平放在地面上，需要4个接触点才能稳定
- 只有1个点会导致物体不断"晃动"

**接触流形 (Contact Manifold)** 就是从碰撞信息中生成最多4个
稳定接触点的过程。

## 二、整体流程

```
GJK/EPA 输出:
    contactPoint1   (body1上的接触点)
    contactPoint2   (body2上的接触点)  
    penetrationAxis (穿透方向/碰撞法线)

流程:
    1. 获取两体各自的 Supporting Face (朝碰撞法线方向的面)
    2. Clip face2 against face1 (多边形裁剪)
    3. 将裁剪结果投影到 face1 平面上
    4. 如果接触点 > 4，PruneContactPoints 裁剪到 ≤ 4
```

## 三、Supporting Face (支撑面)

每个碰撞形状除了 `GetSupport(direction)` 外，
还需提供 `GetSupportingFace(direction)`:

```
// 返回朝 direction 方向最对齐的面的顶点列表
void GetSupportingFace(Vec3 direction, vertices[]);
```

| 形状 | 支撑面 |
|------|--------|
| 球 | 空（永远退化为点接触） |
| 盒 | direction最大分量对应的面 (4个顶点) |
| 凸包 | 法线最接近direction的面 |
| 胶囊 | 如果direction接近轴向→端面圆(边)，否则→空 |

## 四、ClipPolyVsPoly — 多边形裁剪

核心算法: **Sutherland-Hodgman** 逐边裁剪。

```
// ClipPoly.h: ClipPolyVsPoly
输入:
    polygonToClip   — shape2的面 (被裁剪方)
    clippingPolygon — shape1的面 (裁剪方, ≥3顶点)
    clippingNormal  — shape1面法线

算法:
    对 clippingPolygon 的每条边:
        // 计算该边的内法线
        edge = clippingPolygon[(i+1) % n] - clippingPolygon[i]
        clipNormal = clippingNormal × edge  // 指向clippingPolygon内部

        // 用该半平面裁剪当前多边形
        ClipPolyVsPlane(currentPoly, edgeVertex, clipNormal, resultPoly)

        // 如果结果 < 3个顶点，没有交集了
        if resultPoly.size() < 3: return empty

    return resultPoly  // 裁剪后的多边形
```

### ClipPolyVsPlane — 逐边裁剪的单步

```
// 逐顶点遍历多边形，对每条边判断:
for each edge (e1 → e2):
    e1_inside = (planeOrigin - e1) · planeNormal < 0
    e2_inside = (planeOrigin - e2) · planeNormal < 0

    if e1_inside != e2_inside:
        // 一进一出: 计算边与平面的交点
        t = (planeOrigin - e1) · planeNormal / (e2 - e1) · planeNormal
        intersection = e1 + t * (e2 - e1)
        output.add(intersection)

    if e2_inside:
        output.add(e2)   // 内部点保留
```

## 五、ManifoldBetweenTwoFaces — 完整流形生成

```
ManifoldBetweenTwoFaces(point1, point2, axis, maxDist, face1, face2):

    if face1.size ≥ 2 && face2.size ≥ 3:
        // 裁剪 face2 against face1
        if face1.size ≥ 3:
            clipped = ClipPolyVsPoly(face2, face1, axis)
        else if face1.size == 2:
            clipped = ClipPolyVsEdge(face2, face1[0], face1[1], axis)

        // 确定 face1 的平面
        planeOrigin = face1[0]
        planeNormal = (face1[1] - face1[0]) × (face1[2] - face1[0])

        // 将裁剪后的每个点投影到 face1 平面上
        for each p2 in clipped:
            // 沿 penetrationAxis 方向投影
            distance = (p2 - planeOrigin) · planeNormal / (axis · planeNormal)

            // 只保留在阈值内的点
            if distance * |axis| < maxDist:
                p1 = p2 - distance * axis  // 对应的face1上的点
                contacts.add(p1, p2)

    // 如果没有裁剪到任何点，退回到GJK/EPA的单点
    if contacts.empty():
        contacts.add(point1, point2)
```

## 六、PruneContactPoints — 精简到 ≤ 4 点

当裁剪产生 > 4 个接触点时，需要精简。
Jolt 使用**最大面积启发式**:

```
PruneContactPoints(axis, points1, points2):
    // 前提: 接触点 > 4 个

    // 1. 将所有点投影到接触平面上 (去掉法线方向分量)
    projected[i] = points1[i] - (points1[i] · axis) * axis

    // 2. 计算每个点的穿透深度² (作为权重)
    depthSq[i] = |points2[i] - points1[i]|²

    // 3. 选第1个点: 距离质心最远 × 穿透最深 (启发式)
    point1 = argmax_i( |projected[i]|² * depthSq[i] )

    // 4. 选第2个点: 离 point1 最远 × 穿透最深
    point2 = argmax_i( |projected[i] - projected[point1]|² * depthSq[i] )

    // 5. 选第3、4个点: 在 point1-point2 连线两侧各找最远的
    perp = (projected[point2] - projected[point1]) × axis
    point3 = 在 perp 负侧最远的点
    point4 = 在 perp 正侧最远的点

    // 结果: 1~4 个点，构成最大面积四边形
```

**关键设计**: 用 `|距离|² × |穿透深度|²` 作为启发式权重，
确保同时选择"位置分散"和"穿透深"的点，提高稳定性。

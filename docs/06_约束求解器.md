# Jolt 刚体物理核心算法 — 约束求解器 (AxisConstraintPart)

对应 Jolt 源码: `Physics/Constraints/ConstraintPart/AxisConstraintPart.h`
和 `Physics/Constraints/ConstraintPart/SpringPart.h`。

参考文献: "Constraints Derivation for Rigid Body Simulation in 3D" - Daniel Chappuis

## 一、约束求解的数学框架

### 核心问题
物理引擎需要满足各种约束 (不穿透、摩擦、关节等)。
每个约束表示为一个方程 `C(x) = 0` 或不等式 `C(x) ≥ 0`。

对速度层面，约束变为:
```
dC/dt = J · v + b = 0
```

其中:
- `J` = 雅可比矩阵 (约束对广义速度的偏导)
- `v` = 广义速度向量 [v1, ω1, v2, ω2]
- `b` = 偏置项 (bias)

### 求解方法: 顺序冲量 (Sequential Impulse)

对每个约束，计算一个冲量 λ 使约束被满足:

```
λ = -K^{-1} (J · v + b)
```

其中 K (有效质量的逆) 为:
```
K = J · M^{-1} · J^T
```

M^{-1} 是广义逆质量矩阵 = diag(1/m1, I1^{-1}, 1/m2, I2^{-1})。

然后用 λ 更新速度:
```
v' = v + M^{-1} · J^T · λ
```

## 二、AxisConstraintPart — 1-DOF 约束

这是 Jolt 中**最核心的约束求解单元**。
碰撞法线约束和摩擦约束都是它的实例。

### 约束方程

沿轴 n (单位向量) 的距离约束:
```
C = (p2 - p1) · n
```

### 雅可比矩阵 J

```
J = [-n^T,  -(r1+u)×n)^T,  n^T,  (r2×n)^T]
```

其中:
- `n` = 约束轴 (归一化)
- `p1, p2` = 约束点 (世界空间)
- `r1 = p1 - x1` (body1质心到约束点)
- `r2 = p2 - x2` (body2质心到约束点)
- `u = x2 + r2 - x1 - r1 = p2 - p1`
- `x1, x2` = body1、body2 的质心

### 有效质量 K

```
K = J · M^{-1} · J^T

展开:
K = 1/m1 + ((r1+u)×n)^T · I1^{-1} · ((r1+u)×n)
  + 1/m2 + (r2×n)^T · I2^{-1} · (r2×n)
```

这是一个标量 (1×1 矩阵)，因为是 1-DOF 约束。

### CalculateConstraintProperties (预计算)

```
CalculateConstraintProperties(body1, r1PlusU, body2, r2, axis, bias):
    // 缓存中间量
    r1PlusU_cross_axis = (r1+u) × axis
    r2_cross_axis      = r2 × axis

    // body1如果是Dynamic:
    invI1_r1PlusU_cross_axis = I1_world_inv * r1PlusU_cross_axis
    K += 1/m1 + invI1_r1PlusU_cross_axis · r1PlusU_cross_axis

    // body2如果是Dynamic:
    invI2_r2_cross_axis = I2_world_inv * r2_cross_axis
    K += 1/m2 + invI2_r2_cross_axis · r2_cross_axis

    effectiveMass = 1.0 / K    // 有效质量
```

### SolveVelocityConstraint (每次迭代调用)

```
SolveVelocityConstraint(body1, body2, axis, minLambda, maxLambda):
    // 1. 计算 J·v (约束违反速度)
    jv = axis · (v1 - v2)                          // 线速度部分
    jv += r1PlusU_cross_axis · ω1                   // body1角速度部分
    jv -= r2_cross_axis · ω2                         // body2角速度部分

    // 2. 计算拉格朗日乘子
    // λ = -K^{-1} (Jv + b)
    // 其中 b 已被 SpringPart 处理 (见下文)
    lambda = effectiveMass * (jv - bias)

    // 3. 累积并钳制
    newTotalLambda = totalLambda + lambda
    newTotalLambda = clamp(newTotalLambda, minLambda, maxLambda)
    deltaLambda = newTotalLambda - totalLambda
    totalLambda = newTotalLambda

    // 4. 施加冲量
    // body1:
    v1 -= (deltaLambda / m1) * axis
    ω1 -= deltaLambda * invI1_r1PlusU_cross_axis

    // body2:
    v2 += (deltaLambda / m2) * axis
    ω2 += deltaLambda * invI2_r2_cross_axis
```

**关键**: 钳制是对**累积冲量 totalLambda** 而非增量 deltaLambda 做的。
这是 Erin Catto 的 **Accumulated Impulse Clamping** 方法，
比直接钳制增量收敛性好得多。

## 三、SpringPart — 软约束

对应 Erin Catto GDC 2011: "Soft Constraints: Reinventing The Spring"。

### 硬约束
```
bias = baumgarte_bias  // 直接用 Baumgarte 位置修正
softness = 0
effectiveMass = 1/K    // 不变
```

### 软约束 (弹簧)
```
// 输入: stiffness k, damping c, constraint_value C, dt

// softness (gamma): 弱化约束的程度
softness = 1.0 / (dt * (c + dt * k))

// bias: 弹簧驱动力
bias = original_bias + dt * k * softness * C

// 更新有效质量
effectiveMass = 1.0 / (K + softness)
```

然后在 SolveVelocityConstraint 中:
```
// bias计算变为:
actual_bias = bias + softness * totalLambda
lambda = effectiveMass * (jv - actual_bias)
```

`softness * totalLambda` 这项是**隐式阻尼**——
即使 damping ratio=0 也有内建阻尼 (隐式欧拉的特性)。

### 频率/阻尼比 → 刚度/阻尼系数 转换

```
omega = 2 * PI * frequency           // 角频率
k = effectiveMass * omega²           // 刚度
c = 2 * effectiveMass * damping * omega  // 阻尼系数
```

## 四、Warm Starting (热启动)

每帧开始时，用上一帧的 λ 作为初始值:

```
WarmStart(body1, body2, axis, warmStartRatio):
    totalLambda *= warmStartRatio    // ratio = dt_new / dt_old
    deltaLambda = totalLambda        // 整个累积量作为初始冲量施加

    // 施加冲量 (与 SolveVelocityConstraint 中的Step4相同)
    v1 -= (deltaLambda / m1) * axis
    ω1 -= deltaLambda * invI1_r1PlusU_cross_axis
    v2 += (deltaLambda / m2) * axis
    ω2 += deltaLambda * invI2_r2_cross_axis
```

Warm Starting 让求解器从一个已经接近正确的状态开始迭代，
**显著减少所需迭代次数** (通常从 50+ 降到 10)。

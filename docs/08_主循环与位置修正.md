# Jolt 刚体物理核心算法 — 主循环与位置修正

## 一、PhysicsSystem::Update 全局流程

Jolt 的主循环在 `PhysicsSystem::Update(dt, collisionSteps, ...)` 中。
剥去线程调度和 Island 分组后，核心步骤如下:

```
PhysicsSystem::Update(dt):
    // ===== 阶段 0: 外部力 & 速度积分 =====
    for each active body:
        ApplyGravity(body, gravity, dt)
        body.motionProperties.ApplyForceTorqueAndDrag(dt)
        // 此时 body 的 linearVelocity/angularVelocity 已更新
        // 但 position/rotation 还没更新

    // ===== 阶段 1: 宽相 (Broadphase) =====
    FindCollidingPairs()   // AABB 交叉检测
    // 输出: 潜在碰撞对列表

    // ===== 阶段 2: 窄相 (Narrowphase) =====
    for each pair:
        GJK + EPA → 碰撞法线、穿透深度、接触点
        ClipPoly → 接触流形 (≤4点)
    // 输出: ContactConstraint 列表

    // ===== 阶段 3: Island 构建 =====
    // 把通过接触/关节相连的刚体分成独立的 island
    // 每个 island 独立求解 → 更好的并行 & 更快的收敛

    // ===== 阶段 4: 约束求解 =====
    for each island:
        // ---- Warm Start ----
        for each constraint:
            constraint.WarmStart(dt / prev_dt)

        // ---- 速度迭代 ----
        for iter = 0 to numVelocitySteps-1:  // 默认10次
            for each joint_constraint:
                joint.SolveVelocityConstraint(dt)
            for each contact_constraint:
                sSolveVelocityConstraint(contact, mp1, mp2)

    // ===== 阶段 5: 位置积分 =====
    for each active body:
        body.position += body.linearVelocity * dt
        body.rotation = AddRotationStep(body.rotation, body.angularVelocity * dt)
        // AddRotationStep: 轴角 → 四元数增量

    // ===== 阶段 6: 位置修正 (可选，默认2次) =====
    for iter = 0 to numPositionSteps-1:
        for each contact_constraint:
            SolvePositionConstraint(contact)
        for each joint_constraint:
            joint.SolvePositionConstraint(dt)

    // ===== 阶段 7: 睡眠检测 =====
    for each island:
        CheckSleepStatus(island)
```

## 二、关键: 半隐式 (Symplectic) 欧拉 vs 显式欧拉

```
// 显式欧拉 (不稳定):
x += v * dt
v += a * dt

// 半隐式欧拉 (Jolt 使用):
v += a * dt     // 先更新速度 (阶段0)
x += v * dt     // 用新速度更新位置 (阶段5)
```

两者只差顺序，但半隐式欧拉保辛 (symplectic)，
能量误差不会单调积累，长时间仿真也稳定。

## 三、位置修正 — Baumgarte 稳定化

### 问题

速度约束只保证"不再穿透更深"，但已存在的穿透不会消失。
需要额外施加位置修正来推开重叠物体。

### Baumgarte 方法

```
// PhysicsSettings.h:
//   mBaumgarte = 0.2 (法线方向穿透修正比例)
//   mLinearCastThreshold = 0.75 (穿透过深时改用线性投射)

position_error = penetration_depth  // > 0 表示穿透

if position_error > 0:
    // 修正量 = β * 穿透量
    correction = β * position_error

    // 按有效质量分配
    delta_x1 = -correction * normal * (m2 / (m1 + m2))
    delta_x2 = +correction * normal * (m1 / (m1 + m2))

    // 实际代码中更精确: 考虑旋转惯量
    // 通过类似 AxisConstraintPart 的 K 来分配
```

### 位置修正的 clamping

```
// 不设上限修正量，只修正:
//   max(0, penetration - slop)
// 即留一点 "slop" (mPenetrationSlop, 默认极小) 允许的穿透量
// 避免 jitter (物体在穿透边界反复弹跳)
```

### 位置迭代次数

```
PhysicsSettings:
    mNumPositionSteps = 2   // 默认2次位置迭代
    // 通常2次已足够把穿透推到视觉可接受范围
    // 增加到4次可在高质量堆叠中获得更好的稳定性
```

## 四、睡眠 (Sleep / Deactivation)

### 判定条件

```
// 当一个 island 中所有刚体满足:
//   |linearVelocity| < mPointVelocitySleepThreshold (默认极小)
//   |angularVelocity| < mPointVelocitySleepThreshold

// 且持续时间 > mTimeBeforeSleep (默认 0.5 秒)
// → 整个 island 进入睡眠
```

### 睡眠行为

```
// 睡眠的物体:
// 1) 不参与速度积分 (不施加重力)
// 2) 不参与约束求解
// 3) AABB 不更新 (但保留在宽相中)
// 4) 当新物体与之发生碰撞时被唤醒

// 这极大节省性能: 场景中通常 >80% 物体在睡眠
```

## 五、PhysicsSettings 关键常量速查

| 常量 | 默认值 | 含义 |
|------|--------|------|
| `mNumVelocitySteps` | 10 | 速度约束求解迭代次数 |
| `mNumPositionSteps` | 2 | 位置修正迭代次数 |
| `mBaumgarte` | 0.2 | 穿透修正比例 (0~1) |
| `mMinVelocityForRestitution` | 1.0 m/s | 恢复触发阈值 |
| `mPenetrationSlop` | 0.02 | 允许穿透余量 (米) |
| `mTimeBeforeSleep` | 0.5 s | 进入睡眠前的低速持续时间 |
| `mPointVelocitySleepThreshold` | ~0.03 m/s | 睡眠速度阈值 |
| `mSpeculativeContactDistance` | 0.02 | 投机接触距离 |
| `mMaxPenetrationDistance` | 0.04 | 穿透超过此值使用线性投射 |
| `mBodyPairCacheMaxDeltaPositionSq` | (0.001)² | 流形缓存失效位移阈值 |

## 七、需注意的细节

### 重力不在约束前施加的"陷阱"

Jolt 在帧开头先把重力加到速度上。这意味着:
- 静止在地面上的物体帧初会有向下的速度 `g*dt`
- 然后约束求解器会产生法线冲量消除这个速度
- 因此恢复公式中要减去 `g*dt·n` (见 07 文件)

### 位置积分在约束求解之后

关键设计: 位置在约束求解器修正完速度后才积分。
这保证物体移动方向是"经过约束修正的"，
而不是"先移动到错误位置再推回来"。

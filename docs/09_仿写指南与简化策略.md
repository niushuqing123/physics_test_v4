# Jolt 刚体物理核心算法 — 仿写指南与简化策略

## 一、目标回顾

从 Jolt 的 ~10万行 C++ 工业级物理引擎中，提取核心算法，
写成**单文件、≤2000行**的简单刚体物理引擎。
支持: 凸体碰撞、摩擦、恢复、堆叠。

## 二、保留什么 / 砍掉什么

### ✅ 必须保留 (核心算法)

| 模块 | 理由 | 简化方式 |
|------|------|----------|
| 半隐式欧拉积分 | 稳定性基础 | 直接用，几行代码 |
| GJK | 凸体碰撞检测核心 | 保留完整算法，去掉SIMD |
| EPA | 穿透深度 | 保留但限制最大迭代(32) |
| 接触流形 (Clip) | 多点接触 = 稳定堆叠 | Sutherland-Hodgman，保留 |
| Sequential Impulse | 约束求解核心 | AxisConstraintPart 逻辑 |
| 摩擦 (Coulomb) | 物理真实感 | 圆形摩擦锥 |
| Warm Starting | 收敛速度 ×5 | 必须保留 |
| Baumgarte 位置修正 | 修穿透 | β=0.2, 简单几行 |

### ❌ 可以砍掉 (工业包装)

| 模块 | 理由 |
|------|------|
| 多线程 / Job 系统 | 单线程足够教学 |
| Island 分组 | 所有物体放一个 island |
| Broadphase (BVH) | 用 O(n²) 简单遍历 |
| 形状层次结构 | 只支持 Box + Sphere |
| 内存池 / 分配器 | 直接 new/delete 或栈 |
| Contact Cache / Manifold Cache | 每帧重新检测 |
| CCD (连续碰撞) | 用离散就行 |
| 角色控制器 | 与算法无关 |
| Soft body / Vehicle | 超出范围 |
| 压缩存储 (Half-float等) | 直接用 float |

### ⚠️ 可选 (看目标)

| 模块 | 建议 |
|------|------|
| 睡眠 | V1 不要，V2 加上 |
| 陀螺力矩 | V1 跳过，旋转物体少时影响小 |
| 关节约束 | V1 只做接触，V2 加铰链/球窝 |
| 投机接触 | V1 不要，高速场景才需要 |
| Soft constraint (Spring) | 帧率稳定时可不要 |

## 三、推荐数据结构

```cpp
struct Vec3 { float x, y, z; };
struct Quat { float x, y, z, w; };
struct Mat3 { Vec3 col[3]; };  // 列优先 3×3

struct RigidBody {
    // 基本属性
    Vec3  position;
    Quat  rotation;
    Vec3  linearVelocity;
    Vec3  angularVelocity;

    // 质量
    float invMass;              // 0 = 静态物体
    Vec3  invInertiaDiag;       // 主轴惯性张量倒数
    Quat  inertiaRotation;      // 主轴到局部坐标的旋转

    // 材质
    float friction;
    float restitution;

    // 形状
    enum ShapeType { BOX, SPHERE } shapeType;
    Vec3 halfExtent;  // Box 用
    float radius;     // Sphere 用
};

struct ContactPoint {
    Vec3  worldPos;       // 接触世界坐标
    Vec3  normal;         // body1 → body2 方向
    float penetration;    // > 0 穿透
    Vec3  r1, r2;         // 质心到接触点

    // 约束求解用
    float normalLambda;   // 累积法线冲量
    float friction1Lambda;
    float friction2Lambda;
    float effectiveMass;  // 法线方向有效质量
    float effectiveMassT1, effectiveMassT2; // 摩擦有效质量
    float bias;           // 恢复 + 穿透修正 bias
};

struct ContactManifold {
    int body1, body2;
    int numContacts;
    ContactPoint contacts[4];
    Vec3 tangent1, tangent2;   // 切线方向 (全流形共享)
    float combinedFriction;
    float combinedRestitution;
};
```

## 四、推荐实现顺序

### Phase 1: 掉落与地面 (最小可用)

```
1. Vec3/Quat/Mat3 基础数学 (~150行)
2. RigidBody 结构体
3. 半隐式欧拉积分 (重力 + 阻尼)
4. Sphere vs 无限平面碰撞 (解析解, 不需要GJK)
5. 单点接触约束求解 (AxisConstraintPart 简化版)
6. Baumgarte 位置修正
→ 球掉到地上并静止
```

### Phase 2: 凸体碰撞 (~+500行)

```
7. GJK (支持点 + 迭代)
8. EPA (基本版)
9. Box 的 Support 函数
10. Box vs Box 碰撞
→ BOX 掉到地上
```

### Phase 3: 接触流形 + 摩擦 (~+300行)

```
11. GetSupportingFace (Box 实现)
12. ClipPolyVsPlane + ClipPolyVsPoly
13. PruneContactPoints (≤4点)
14. 摩擦约束 (圆形 Coulomb 锥)
15. Warm Starting
→ BOX 堆叠稳定
```

### Phase 4: 完善 (~+200行)

```
16. Sphere 的 GJK support
17. 多物体场景
18. 睡眠
19. 简单渲染
```

## 五、关键简化的代价

| 简化项 | 影响 |
|--------|------|
| 无 Broadphase | N 物体 → O(N²) 碰撞检测，50+ 物体会慢 |
| 无 Island | 所有物体一起解，不能并行 |
| 无 CCD | 高速小物体可能穿过薄墙 |
| 无 Contact Cache | 每帧重新 GJK+EPA，但 warm start 的 λ 仍需缓存 |
| 无 Soft Constraint | 时间步过大时约束会感觉"硬"，帧率不稳时可能弹跳 |
| 无陀螺力矩 | 陀螺/飞盘旋转不真实，对一般场景无感 |

## 六、Support 函数速查

```cpp
// 球体
Vec3 SupportSphere(Vec3 dir, float radius) {
    return normalize(dir) * radius;
}

// Box (中心在原点，halfExtent = h)
Vec3 SupportBox(Vec3 dir, Vec3 h) {
    return Vec3(
        dir.x >= 0 ? h.x : -h.x,
        dir.y >= 0 ? h.y : -h.y,
        dir.z >= 0 ? h.z : -h.z
    );
}

// 世界坐标下的 Support (通用)
Vec3 SupportWorld(Body& body, Vec3 worldDir) {
    Vec3 localDir = body.rotation.Inverse() * worldDir;
    Vec3 localPt = SupportLocal(body, localDir);  // Box/Sphere
    return body.rotation * localPt + body.position;
}

// Minkowski 差 A-B 的 Support
Vec3 SupportMinkowski(Body& A, Body& B, Vec3 dir) {
    return SupportWorld(A, dir) - SupportWorld(B, -dir);
}
```

## 七、约束求解器模板

```cpp
// 通用 1-DOF 约束求解 (法线/摩擦都用)
float SolveConstraint1D(
    RigidBody& b1, RigidBody& b2,
    Vec3 r1, Vec3 r2,      // 质心到接触点
    Vec3 axis,              // 约束轴
    float bias,             // 目标速度偏移
    float& totalLambda,     // 累积冲量 (in/out)
    float minLambda, float maxLambda,
    float effectiveMass     // 预计算的 K⁻¹
) {
    // 计算约束速度
    Vec3 v1 = b1.linearVelocity + cross(b1.angularVelocity, r1);
    Vec3 v2 = b2.linearVelocity + cross(b2.angularVelocity, r2);
    float jv = dot(v2 - v1, axis);

    // 计算冲量增量
    float deltaLambda = effectiveMass * (jv - bias);

    // 累积冲量钳制
    float oldLambda = totalLambda;
    totalLambda = clamp(totalLambda + deltaLambda, minLambda, maxLambda);
    deltaLambda = totalLambda - oldLambda;

    // 施加冲量
    Vec3 impulse = axis * deltaLambda;
    b1.linearVelocity  -= impulse * b1.invMass;
    b1.angularVelocity -= b1.GetWorldInvInertia() * cross(r1, impulse);
    b2.linearVelocity  += impulse * b2.invMass;
    b2.angularVelocity += b2.GetWorldInvInertia() * cross(r2, impulse);

    return deltaLambda;
}

// 有效质量预计算
float ComputeEffectiveMass(
    RigidBody& b1, RigidBody& b2,
    Vec3 r1, Vec3 r2, Vec3 axis
) {
    // K = m1⁻¹ + m2⁻¹ + (I1⁻¹(r1×n))×r1 · n + (I2⁻¹(r2×n))×r2 · n
    float k = b1.invMass + b2.invMass;
    Vec3 t1 = cross(b1.GetWorldInvInertia() * cross(r1, axis), r1);
    Vec3 t2 = cross(b2.GetWorldInvInertia() * cross(r2, axis), r2);
    k += dot(t1 + t2, axis);
    return 1.0f / k;
}
```

## 九、语言移植建议

| 目标语言 | 建议 |
|----------|------|
| C | 结构体 + 函数，Vec3/Quat 不重载运算符就用宏或函数 |
| Rust | 用 glam 或 nalgebra 做数学，约束用 struct + impl |
| Python | numpy 做向量运算，速度够用于教学 |
| JS/TS | 不建议用 class，plain object + 函数更清晰 |
| GLSL/WebGPU | 可做 GPU 粒子物理，但约束求解不适合 GPU |

# Jolt 布娃娃系统调查报告 — Part 1: 概述与文件地图

## 一、什么是布娃娃 (Ragdoll)

布娃娃是一种用**多个刚体 + 关节约束**来模拟角色身体物理行为的系统。与纯动画驱动的角色不同，布娃娃的每个肢体都是独立的物理刚体，通过约束连接，能自然地响应碰撞、重力和外力。

典型应用场景：
- 角色死亡后的自然倒地
- 被爆炸/冲击波击飞
- 物理驱动的动画混合（powered ragdoll）
- 运动学驱动（kinematic ragdoll）推动场景物体

## 二、Jolt 布娃娃的架构层级

| 层级 | 组成 | 说明 |
|------|------|------|
| **数据定义层** | `Skeleton`, `SkeletonPose`, `SkeletalAnimation` | 骨骼拓扑、关节姿态、动画采样 |
| **配置层** | `RagdollSettings`, `RagdollSettings::Part` | 每个肢体的形状/质量/约束参数 |
| **约束层** | `SwingTwistConstraint` (主要), 也可用 `HingeConstraint` 等 | 关节旋转限制 + 电机驱动 |
| **碰撞过滤层** | `GroupFilterTable` | 禁用父子/重叠肢体之间的碰撞 |
| **运行时层** | `Ragdoll` | 管理刚体+约束的生命周期、姿态驱动 |
| **映射层** | `SkeletonMapper` | 低精度骨骼(布娃娃) ↔ 高精度骨骼(动画) 映射 |

## 三、核心文件地图

所有路径相对于 `build/_deps/joltphysics-src/`：

### 3.1 骨骼系统 (`Jolt/Skeleton/`)

| 文件 | 作用 |
|------|------|
| `Skeleton.h/.cpp` | 骨骼拓扑定义：关节名称、父子关系（数组+parentIndex） |
| `SkeletonPose.h/.cpp` | 骨骼姿态实例：每个关节的局部 JointState（平移+旋转） + 世界空间 JointMatrix |
| `SkeletonMapper.h/.cpp` | 低精度布娃娃骨骼 ↔ 高精度动画骨骼之间的关节映射 |

### 3.2 布娃娃核心 (`Jolt/Physics/Ragdoll/`)

| 文件 | 作用 |
|------|------|
| `Ragdoll.h` | `RagdollSettings`（配置）+ `Ragdoll`（运行时）类定义 |
| `Ragdoll.cpp` | 创建、添加/移除物理系统、姿态设置/获取、运动学/电机驱动、稳定化算法 |

### 3.3 关节约束 (`Jolt/Physics/Constraints/`)

| 文件 | 作用（与布娃娃相关的） |
|------|------|
| `SwingTwistConstraint.h/.cpp` | **布娃娃首选约束**：摆动+扭转限制 + 电机 |
| `ConstraintPart/SwingTwistConstraintPart.h` | 底层数学：分解swing/twist、锥形/金字塔限制、夹紧 |
| `ConstraintPart/PointConstraintPart.h` | 点约束部件（SwingTwist内部用于位置锁定） |
| `ConstraintPart/AngleConstraintPart.h` | 角度约束部件（SwingTwist内部用于电机） |
| `MotorSettings.h` | 电机参数：弹簧设置、力/扭矩限制 |
| `HingeConstraint.h` | 铰链约束（可替代SwingTwist用于肘/膝） |
| `ConeConstraint.h` | 锥形约束（简化版SwingTwist） |
| `FixedConstraint.h` | 固定约束（调试用） |
| `PointConstraint.h` | 点约束（只锁位置不限制旋转） |

### 3.4 碰撞过滤 (`Jolt/Physics/Collision/`)

| 文件 | 作用 |
|------|------|
| `GroupFilterTable.h` | N×N碰撞过滤表，用于禁用布娃娃肢体间碰撞 |

### 3.5 示例代码 (`Samples/` 和 `PerformanceTest/`)

| 文件 | 作用 |
|------|------|
| `Samples/Utils/RagdollLoader.h/.cpp` | 从文件加载布娃娃 / 从代码创建布娃娃的完整示例 |
| `Samples/Tests/Rig/PoweredRigTest.cpp` | 电机驱动布娃娃跟随动画 |
| `Samples/Tests/Rig/KinematicRigTest.cpp` | 运动学驱动布娃娃 |
| `Samples/Tests/Rig/RigPileTest.cpp` | 布娃娃堆叠测试 |
| `Samples/Tests/Rig/CreateRigTest.cpp` | 从代码创建布娃娃 |
| `Samples/Tests/Rig/SkeletonMapperTest.cpp` | 骨骼映射测试 |
| `PerformanceTest/RagdollScene.h` | 性能测试：大量布娃娃堆叠 |

## 四、布娃娃系统的核心数据流

```
                创建阶段
┌─────────────────────────────────────────────┐
│  Skeleton (骨骼拓扑)                          │
│       ↓                                      │
│  RagdollSettings                             │
│    ├─ Part[0..N]: BodyCreationSettings       │
│    │   ├─ Shape (CapsuleShape等)             │
│    │   ├─ position, rotation                 │
│    │   ├─ motionType (Dynamic/Kinematic)     │
│    │   └─ mToParent: SwingTwistConstraint    │
│    ├─ AdditionalConstraints (可选)            │
│    ├─ Stabilize() → 平衡质量/惯性比           │
│    └─ DisableParentChildCollisions()         │
│       ↓                                      │
│  CreateRagdoll() → Ragdoll 运行时实例         │
│    ├─ BodyID[] (每个肢体一个Body)             │
│    └─ TwoBodyConstraint[] (关节约束)          │
└─────────────────────────────────────────────┘

                运行阶段
┌─────────────────────────────────────────────┐
│  Ragdoll                                     │
│    ├─ AddToPhysicsSystem() → 激活物理仿真    │
│    ├─ SetPose(SkeletonPose) → 瞬间移动      │
│    ├─ DriveToPoseUsingKinematics() → 运动学  │
│    ├─ DriveToPoseUsingMotors() → 电机驱动    │
│    ├─ GetPose() → 读回当前物理姿态           │
│    └─ RemoveFromPhysicsSystem() → 移除       │
└─────────────────────────────────────────────┘
```

## 五、与刚体物理系统的关系

布娃娃**不是**独立于刚体物理的新系统，而是刚体物理之上的一层**组织管理**：
- 每个肢体就是一个普通的 `Body`（有形状、质量、速度）
- 每个关节就是一个普通的 `TwoBodyConstraint`
- 布娃娃系统只是把它们按树形骨骼结构组织起来，提供便捷的批量操作接口

**这意味着：只要你的物理后端支持刚体 + 约束（尤其是SwingTwist类约束），就可以在其上实现布娃娃。**

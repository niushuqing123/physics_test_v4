# Jolt 布娃娃系统调查报告 — Part 5: 运行时与驱动模式

## 一、Ragdoll 类 — 运行时管理

**源文件**: `Jolt/Physics/Ragdoll/Ragdoll.h` (后半部分), `Ragdoll.cpp`

Ragdoll 类是布娃娃的**运行时实例**，管理一组 BodyID 和 Constraint 的生命周期。

### 1.1 核心数据

```cpp
class Ragdoll {
    RefConst<RagdollSettings>        mRagdollSettings;  // 创建它的配置
    Array<BodyID>                    mBodyIDs;           // 物理系统中的刚体ID列表
    Array<Ref<TwoBodyConstraint>>    mConstraints;       // 约束列表
    PhysicsSystem*                   mSystem;            // 物理系统引用
};
```

- `mBodyIDs` 与 `mRagdollSettings->mParts` 一一对应
- `mConstraints` 包含父子约束 + AdditionalConstraints

## 二、生命周期

### 2.1 创建

```cpp
// 从设置创建（为每个 Part 创建 Body + 为每个约束创建 Constraint）
Ragdoll* ragdoll = settings->CreateRagdoll(
    groupID,      // CollisionGroup::GroupID, 每个布娃娃唯一
    userData,     // 用户自定义数据
    physicsSystem
);
```

`CreateRagdoll()` 内部流程：
1. 遍历 `mParts`，对每个 Part 调用 `BodyInterface::CreateBody()` → 得到 Body*
2. 设置 `CollisionGroup::GroupID`
3. 对有 `mToParent` 的 Part，在子body和父body之间创建约束
4. 处理 `mAdditionalConstraints`

### 2.2 加入物理世界

```cpp
ragdoll->AddToPhysicsSystem(EActivation::Activate);
```

内部：
1. 批量添加所有 Body (`AddBodiesPrepare` + `AddBodiesFinalize`)
2. 批量添加所有 Constraint (`AddConstraints`)

### 2.3 移除

```cpp
ragdoll->RemoveFromPhysicsSystem();
```

内部：先移除约束，再批量移除刚体。

### 2.4 销毁

```cpp
// 析构函数自动销毁所有 Body
~Ragdoll() {
    mSystem->GetBodyInterface().DestroyBodies(mBodyIDs.data(), count);
}
```

## 三、三种姿态驱动模式

### 3.1 瞬间设置 — SetPose

```cpp
void Ragdoll::SetPose(const SkeletonPose& inPose);
void Ragdoll::SetPose(RVec3Arg inRootOffset, const Mat44* inJointMatrices);
```

**机制**: 直接调用 `BodyInterface::SetPositionAndRotation()` 瞬间传送每个刚体。

**用途**: 初始化布娃娃位置、重置姿态。

```cpp
// 内部实现
for (int i = 0; i < bodyCount; ++i) {
    const Mat44& joint = inJointMatrices[i];
    bodyInterface.SetPositionAndRotation(
        mBodyIDs[i],
        rootOffset + joint.GetTranslation(),
        joint.GetQuaternion(),
        EActivation::DontActivate
    );
}
```

### 3.2 运动学驱动 — DriveToPoseUsingKinematics

```cpp
void Ragdoll::DriveToPoseUsingKinematics(const SkeletonPose& inPose, float inDeltaTime);
```

**机制**: 调用 `BodyInterface::MoveKinematic()`，计算所需速度使刚体在 `deltaTime` 内到达目标位置。

**用途**: 运动学驱动的布娃娃（motionType = Kinematic），能推动场景中的其他物体。

```cpp
// 内部实现
for (int i = 0; i < bodyCount; ++i) {
    const Mat44& joint = inJointMatrices[i];
    bodyInterface.MoveKinematic(
        mBodyIDs[i],
        rootOffset + joint.GetTranslation(),
        joint.GetQuaternion(),
        deltaTime
    );
}
```

**使用场景示例** (KinematicRigTest):
```cpp
void PrePhysicsUpdate(float dt) {
    // 1. 采样前一帧的姿态（验证布娃娃是否跟上了）
    animation->Sample(prevTime, pose);
    
    // 2. 采样下一帧的目标姿态
    time += dt;
    animation->Sample(time, pose);
    pose.CalculateJointMatrices();
    
    // 3. 运动学驱动
    ragdoll->DriveToPoseUsingKinematics(pose, dt);
}
```

### 3.3 电机驱动 — DriveToPoseUsingMotors

```cpp
void Ragdoll::DriveToPoseUsingMotors(const SkeletonPose& inPose);
```

**机制**: 为每个 SwingTwistConstraint 设置电机目标朝向，让物理引擎通过弹簧力驱动关节到目标旋转。

**用途**: powered ragdoll，布娃娃尝试跟随动画但仍受物理影响（可被推倒、摔倒）。

```cpp
// 内部实现
for (int i = 0; i < jointCount; ++i) {
    int constraintIdx = mRagdollSettings->GetConstraintIndexForBodyIndex(i);
    if (constraintIdx >= 0) {
        const JointState& jointState = inPose.GetJoint(i);
        TwoBodyConstraint* constraint = mConstraints[constraintIdx];
        
        if (constraint->GetSubType() == EConstraintSubType::SwingTwist) {
            SwingTwistConstraint* st = static_cast<SwingTwistConstraint*>(constraint);
            st->SetSwingMotorState(EMotorState::Position);
            st->SetTwistMotorState(EMotorState::Position);
            st->SetTargetOrientationBS(jointState.mRotation);
            // BS = Body Space, 使用局部空间旋转作为目标
        }
    }
}
```

**使用场景示例** (PoweredRigTest):
```cpp
void PrePhysicsUpdate(float dt) {
    time += dt;
    animation->Sample(time, pose);
    
    // 根关节位置从物理世界读取（物理引擎控制）
    Vec3 rootPos;
    Quat rootRot;
    ragdoll->GetRootTransform(rootPos, rootRot);
    pose.GetJoint(0).mTranslation = Vec3::sZero();
    pose.GetJoint(0).mRotation = rootRot;  // 保持物理旋转
    pose.SetRootOffset(rootPos);
    
    pose.CalculateJointMatrices();
    ragdoll->DriveToPoseUsingMotors(pose);
}
```

## 四、姿态读取

### 4.1 GetPose

```cpp
void Ragdoll::GetPose(SkeletonPose& outPose);
void Ragdoll::GetPose(RVec3& outRootOffset, Mat44* outJointMatrices);
```

从物理世界读取每个刚体的当前变换，写入 JointMatrices：

```cpp
// 根节点：完整世界变换
RMat44 rootTransform = rootBody->GetWorldTransform();
outRootOffset = rootTransform.GetTranslation();
outJointMatrices[0] = Mat44(rootTransform的旋转列, (0,0,0,1));

// 其他节点：相对于根的变换
for (int b = 1; b < bodyCount; ++b) {
    RMat44 transform = body[b]->GetWorldTransform();
    outJointMatrices[b] = Mat44(旋转列, (position - rootOffset, 1));
}
```

### 4.2 GetRootTransform

```cpp
void GetRootTransform(RVec3& outPosition, Quat& outRotation);
```

只获取根节点（索引0）的世界位置和旋转。

## 六、三种模式对比

| 特性 | SetPose | DriveToPoseUsingKinematics | DriveToPoseUsingMotors |
|------|---------|---------------------------|----------------------|
| **Motion Type** | 任意 | Kinematic | Dynamic |
| **移动方式** | 瞬间传送 | 计算速度平滑移动 | 弹簧电机 |
| **物理响应** | 无 | 能推动其他物体 | 完全物理 |
| **跟随精度** | 100% | 高 | 取决于电机强度 |
| **可被外力影响** | 否 | 否 | 是 |
| **典型用途** | 初始化 | NPC行走推箱子 | 被打飞、死亡倒地 |

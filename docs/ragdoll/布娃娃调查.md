# Jolt 布娃娃系统调查报告 — Part 1: 概述与文件地图

## 一、什么是布娃娃 (Ragdoll)

布娃娃是一种用**多个刚体 + 关节约束**来模拟角色身体物理行为的系统。与纯动画驱动的角色不同，布娃娃的每个肢体都是独立的物理刚体，通过约束连接，能自然地响应碰撞、重力和外力。

典型应用场景：
- 角色死亡后的自然倒地
- 被爆炸/冲击波击飞
- 物理驱动的动画混合（powered ragdoll）
- 运动学驱动（kinematic ragdoll）推动场景物体

## 二、Jolt 布娃娃的架构层级

| 层级 | 组成 | 说明 |
|------|------|------|
| **数据定义层** | `Skeleton`, `SkeletonPose`, `SkeletalAnimation` | 骨骼拓扑、关节姿态、动画采样 |
| **配置层** | `RagdollSettings`, `RagdollSettings::Part` | 每个肢体的形状/质量/约束参数 |
| **约束层** | `SwingTwistConstraint` (主要), 也可用 `HingeConstraint` 等 | 关节旋转限制 + 电机驱动 |
| **碰撞过滤层** | `GroupFilterTable` | 禁用父子/重叠肢体之间的碰撞 |
| **运行时层** | `Ragdoll` | 管理刚体+约束的生命周期、姿态驱动 |
| **映射层** | `SkeletonMapper` | 低精度骨骼(布娃娃) ↔ 高精度骨骼(动画) 映射 |

## 五、与刚体物理系统的关系

布娃娃**不是**独立于刚体物理的新系统，而是刚体物理之上的一层**组织管理**：
- 每个肢体就是一个普通的 `Body`（有形状、质量、速度）
- 每个关节就是一个普通的 `TwoBodyConstraint`
- 布娃娃系统只是把它们按树形骨骼结构组织起来，提供便捷的批量操作接口

**这意味着：只要你的物理后端支持刚体 + 约束（尤其是SwingTwist类约束），就可以在其上实现布娃娃。**

# Jolt 布娃娃系统调查报告 — Part 2: 骨骼与姿态系统

## 一、Skeleton — 骨骼拓扑

**源文件**: `Jolt/Skeleton/Skeleton.h/.cpp`

Skeleton 是一个纯数据结构，描述关节的**树形层级关系**，不包含任何位置/旋转信息。

### 1.1 核心数据

```cpp
class Skeleton {
    struct Joint {
        String mName;              // 关节名称 (如 "UpperArmL")
        String mParentName;        // 父关节名称 (如 "UpperBody")
        int    mParentJointIndex;  // 父关节索引 (-1 表示根节点)
    };
    
    Array<Joint> mJoints;          // 所有关节，按拓扑序排列
};
```

### 1.2 关键要求

- **拓扑排序**: 父关节必须在子关节之前出现在数组中（`AreJointsCorrectlyOrdered()` 验证）
- 关节通过 `mParentJointIndex` 形成树结构，根关节的 parentIndex = -1
- `CalculateParentJointIndices()` 根据关节名称自动填充 parentIndex

### 1.3 典型人体骨骼（来自 RagdollLoader::sCreate）

```
LowerBody (root, 无父)
├── MidBody
│   └── UpperBody
│       ├── Head
│       ├── UpperArmL
│       │   └── LowerArmL
│       └── UpperArmR
│           └── LowerArmR
├── UpperLegL
│   └── LowerLegL
└── UpperLegR
    └── LowerLegR
```

共 12 个关节。每个关节在布娃娃中对应一个物理刚体。

## 二、SkeletonPose — 骨骼姿态

**源文件**: `Jolt/Skeleton/SkeletonPose.h/.cpp`

SkeletonPose 是 Skeleton 的一个**实例化姿态**，存储每个关节当前的位置和朝向。

### 2.1 核心数据

```cpp
class SkeletonPose {
    RefConst<Skeleton>  mSkeleton;       // 引用的骨骼定义
    RVec3               mRootOffset;     // 根节点世界空间偏移
    
    // 局部空间关节状态 (相对于父关节)
    Array<JointState>   mJoints;         // 每个关节的局部状态
    
    // 世界空间关节矩阵 (由 CalculateJointMatrices 计算)
    Array<Mat44>        mJointMatrices;  // 每个关节的世界变换矩阵
};

// JointState 定义 (来自 SkeletalAnimation)
struct JointState {
    Vec3  mTranslation;   // 相对于父关节的平移
    Quat  mRotation;      // 相对于父关节的旋转
};
```

### 2.2 两种空间

| 空间 | 数据 | 说明 |
|------|------|------|
| **局部空间** | `mJoints[i]` (JointState) | 每个关节相对于其父关节的平移+旋转 |
| **世界空间** | `mJointMatrices[i]` (Mat44) | 每个关节在世界中的完整变换矩阵 |

### 2.3 关键操作

```
// 局部 → 世界: 沿骨骼树自顶向下累乘
CalculateJointMatrices():
    for each joint i:
        if parent == -1:
            jointMatrix[i] = rootOffset * localTransform(i)
        else:
            jointMatrix[i] = jointMatrix[parent] * localTransform(i)

// 世界 → 局部: 逆操作
CalculateJointStates():
    for each joint i:
        if parent == -1:
            jointState[i] = decompose(jointMatrix[i])
        else:
            jointState[i] = decompose(jointMatrix[parent]^-1 * jointMatrix[i])
```

### 2.4 与布娃娃的关系

布娃娃的 `SetPose()` 和 `GetPose()` 直接使用 SkeletonPose：
- **SetPose**: 读取 `mJointMatrices`，将每个刚体瞬间移到对应矩阵的位置/朝向
- **GetPose**: 读取每个刚体的世界变换，写回 `mJointMatrices`

## 三、SkeletalAnimation — 动画采样

布娃娃常配合动画使用（例如 powered ragdoll 跟随动画姿态），动画通过 `SkeletalAnimation::Sample(time, pose)` 采样到 SkeletonPose 中。

### 3.1 典型工作流

```cpp
// 初始化
SkeletonPose pose;
pose.SetSkeleton(ragdollSettings->GetSkeleton());

// 每帧
animation->Sample(currentTime, pose);   // 采样到局部空间
pose.CalculateJointMatrices();           // 转换到世界空间

// 驱动布娃娃
ragdoll->DriveToPoseUsingMotors(pose);   // 电机驱动到目标姿态
// 或
ragdoll->DriveToPoseUsingKinematics(pose, deltaTime);  // 运动学驱动
```

## 四、仿写要点

### 4.1 最小必要实现

对于你自己的物理后端，骨骼系统需要实现的最少内容：

```
struct Joint {
    std::string name;
    int parentIndex;      // -1 = root
};

struct Skeleton {
    std::vector<Joint> joints;
};

struct JointState {
    Vec3 translation;     // 局部平移
    Quat rotation;        // 局部旋转
};

struct SkeletonPose {
    Skeleton* skeleton;
    Vec3 rootOffset;                    // 根节点世界偏移
    std::vector<JointState> joints;     // 局部空间
    std::vector<Mat4> jointMatrices;    // 世界空间
    
    void CalculateJointMatrices();      // 局部→世界
};
```

### 4.2 简化策略

- **不需要** SkeletalAnimation 动画系统（可以手动设置姿态）
- **不需要** SkeletonMapper（除非你有高精度/低精度两套骨骼）
- **不需要** 二进制序列化
- 只需要 `CalculateJointMatrices()`（从局部空间到世界空间的累乘）

# Jolt 布娃娃系统调查报告 — Part 3: 布娃娃配置与稳定化

## 一、RagdollSettings — 布娃娃配置

**源文件**: `Jolt/Physics/Ragdoll/Ragdoll.h` (前半部分)

RagdollSettings 是布娃娃的**蓝图/模板**，描述所有肢体的形状、质量、约束参数。可以从一个 Settings 创建多个 Ragdoll 实例。

### 1.1 核心结构

```cpp
class RagdollSettings {
    Ref<Skeleton>  mSkeleton;    // 引用的骨骼
    PartVector     mParts;       // 与 skeleton.joints 一一对应
    
    // 额外约束（非父子关系的约束，如手腕绑定等）
    AdditionalConstraintVector mAdditionalConstraints;
};
```

### 1.2 Part — 单个肢体定义

Part **继承自** `BodyCreationSettings`，即包含创建一个刚体所需的全部信息，外加一个连接到父关节的约束设置：

```cpp
class Part : public BodyCreationSettings {
    // 继承自 BodyCreationSettings 的:
    //   Shape*       shape;          // 碰撞形状
    //   RVec3        position;       // 初始位置
    //   Quat         rotation;       // 初始朝向
    //   EMotionType  motionType;     // Dynamic / Kinematic / Static
    //   ObjectLayer  objectLayer;    // 碰撞层
    //   ... 质量、摩擦、弹性 等
    
    Ref<TwoBodyConstraintSettings> mToParent;  // 连接到父关节的约束
    // 根关节(index=0) 的 mToParent 为 nullptr
};
```

### 1.3 AdditionalConstraint — 额外约束

用于非父子关系的关节连接（例如骨盆两侧的交叉约束）：

```cpp
class AdditionalConstraint {
    int mBodyIdx[2];                        // 连接的两个 Part 索引
    Ref<TwoBodyConstraintSettings> mConstraint;  // 约束设置
};
```

## 三、Stabilize() — 稳定化算法

**源文件**: `Ragdoll.cpp` 中的 `RagdollSettings::Stabilize()`

这是来自 Havok 的技术（Oliver Strunk, "Stop my Constraints from Blowing Up!"），解决布娃娃的**约束爆炸**问题。

### 3.1 问题

当父肢体很轻而子肢体很重时（例如手指很重但手腕很轻），约束求解器会变得不稳定，产生剧烈抖动。

### 3.2 算法（两步）

**步骤 1：限制父子质量比**

```
对每条从根到叶的肢体链:
    cMinMassRatio = 0.8
    cMaxMassRatio = 1.2
    
    // 计算每个肢体的目标质量比
    for 每个子肢体 i:
        ratio = mass[child] / mass[parent]
        mass_ratios[i] = mass_ratios[parent] * clamp(ratio, 0.8, 1.2)
    
    // 重新分配质量（保持总质量不变）
    totalMass = sum(所有肢体质量)
    ratio_to_mass = totalMass / sum(mass_ratios)
    
    for 每个肢体 i:
        new_mass = mass_ratios[i] * ratio_to_mass
        inertia *= new_mass / old_mass   // 惯性等比缩放
        mass = new_mass
```

**步骤 2：增大父关节惯性**

```
cMaxInertiaIncrease = 2.0

// 对每个肢体，计算其所有子肢体的惯性之和
for 每个肢体 i (从叶到根遍历):
    childSum[parent] += principalInertia[i].x + childSum[i]

// 确保父关节惯性 ≥ 子关节惯性之和（有上限）
for 每个肢体 i:
    if childSum[i] > 0:
        minimum = min(2.0 * principalInertia[i].x, childSum[i])
        principalInertia[i] = max(principalInertia[i], minimum)
```

### 3.3 要点

- 保持**总质量守恒**（只做重新分配）
- 通过 `MassProperties::DecomposePrincipalMomentsOfInertia()` 分解惯性张量
- Static 刚体不参与质量重分配

## 四、DisableParentChildCollisions() — 碰撞过滤

### 4.1 基本过滤

创建 `GroupFilterTable(jointCount)`，对每对父子关节禁用碰撞：

```cpp
for each joint:
    if has parent:
        groupFilter->DisableCollision(jointIdx, parentJointIdx)
```

### 4.2 基于几何的自动过滤

如果提供了初始姿态（`inJointMatrices`），还会自动检测哪些非父子关节在初始状态下已经重叠，并禁用它们之间的碰撞：

```cpp
for each pair (j1, j2):
    if collision still enabled:
        // 用 CollisionDispatch 检测两个 Shape 是否重叠
        if shapes overlap (with separation < minSeparationDistance):
            groupFilter->DisableCollision(j1, j2)
```

### 4.3 分组

每个 Part 被赋予：
- `SubGroupID` = 关节索引
- `GroupFilter` = 共享的 GroupFilterTable

在 `CreateRagdoll()` 时设置 `GroupID`（每个布娃娃实例一个唯一ID），这样**不同布娃娃之间仍然碰撞**，只有同一布娃娃内的特定肢体不碰撞。

# Jolt 布娃娃系统调查报告 — Part 4: SwingTwist 约束详解

## 一、为什么是 SwingTwistConstraint

布娃娃关节本质上是**球窝关节**（ball-and-socket joint），需要限制：
1. **摆动 (Swing)**: 骨骼偏离其自然轴线的旋转（如手臂侧举、前抬）
2. **扭转 (Twist)**: 骨骼沿自身轴线的旋转（如前臂旋转手腕）

`SwingTwistConstraint` 正是为此设计的专用约束，是 Jolt 布娃娃的**首选约束类型**。

**源文件**: 
- `Jolt/Physics/Constraints/SwingTwistConstraint.h/.cpp`
- `Jolt/Physics/Constraints/ConstraintPart/SwingTwistConstraintPart.h`

---

## 二、约束参数

### 2.1 SwingTwistConstraintSettings

```cpp
struct SwingTwistConstraintSettings {
    // === 参考帧（关节位置和轴向）===
    EConstraintSpace  mSpace;      // WorldSpace 或 LocalSpace
    RVec3  mPosition1, mPosition2; // body1/body2 上的约束锚点
    Vec3   mTwistAxis1, mTwistAxis2; // 扭转轴（沿骨骼延伸方向）
    Vec3   mPlaneAxis1, mPlaneAxis2; // 平面轴（与twist轴垂直的参考方向）
    
    // === 摆动限制类型 ===
    ESwingType  mSwingType;  // Cone (锥形) 或 Pyramid (金字塔形)
    
    // === 摆动角度限制 ===
    float  mNormalHalfConeAngle;  // 法线方向半锥角 (rad)
    float  mPlaneHalfConeAngle;   // 平面方向半锥角 (rad)
    
    // === 扭转角度限制 ===
    float  mTwistMinAngle;        // 最小扭转角 (rad), ∈ [-π, π]
    float  mTwistMaxAngle;        // 最大扭转角 (rad), ∈ [-π, π]
    
    // === 摩擦 ===
    float  mMaxFrictionTorque;    // 无电机时的摩擦扭矩 (N·m)
    
    // === 电机设置 ===
    MotorSettings  mSwingMotorSettings;  // 摆动电机
    MotorSettings  mTwistMotorSettings;  // 扭转电机
};
```

### 2.2 三轴示意

```
         PlaneAxis (Z)
              |
              |  NormalHalfConeAngle
              |  ↕
    ----------+----------→ TwistAxis (X) ← 骨骼延伸方向
              |  ↕
              |  PlaneHalfConeAngle
              |
         NormalAxis (Y) = PlaneAxis × TwistAxis
```

- **TwistAxis**: 沿骨骼延伸的方向，扭转围绕此轴
- **PlaneAxis**: 约束参考平面的法线方向
- **NormalAxis**: 自动计算 = PlaneAxis × TwistAxis

---

## 三、内部约束结构

SwingTwistConstraint 内部由**三个约束部件**组成：

```cpp
class SwingTwistConstraint {
    // 1. 位置约束：锁定两个锚点位置
    PointConstraintPart      mPointConstraintPart;     // 3-DOF 位置
    
    // 2. 摆动+扭转限制
    SwingTwistConstraintPart mSwingTwistConstraintPart; // 角度限制
    
    // 3. 电机约束（3轴）
    AngleConstraintPart      mMotorConstraintPart[3];   // twist + swing_y + swing_z
};
```

### 3.1 约束空间内的旋转分解

关键数学：将 body2 相对于 body1 的旋转分解为 swing 和 twist：

```
q = (body1_rotation * constraintToBody1)^-1 * (body2_rotation * constraintToBody2)
```

然后将 `q` 分解为：
```
q = q_swing * q_twist
```
其中：
- `q_twist` 只有 x 分量 + w（绕 X 轴旋转）
- `q_swing` 只有 y, z 分量 + w（绕 Y, Z 轴旋转）

---

## 四、摆动限制的两种模式

### 4.1 Cone（锥形）— 默认

将 `(q_swing.y, q_swing.z)` 看作椭圆上的点：

$$\frac{q_{swing}.y^2}{\sin^2(\theta_{swingY}/2)} + \frac{q_{swing}.z^2}{\sin^2(\theta_{swingZ}/2)} \leq 1$$

如果在椭圆外，投影到椭圆边界上的最近点。

**优点**: 平滑的锥形限制空间
**缺点**: 大角度时锥形会变形

### 4.2 Pyramid（金字塔形）

将 swing 分解为绕 Y 和 Z 轴的独立旋转角度：

```
halfAngleY = atan2(q_swing.y, q_swing.w)
halfAngleZ = atan2(q_swing.z, q_swing.w)
```

各自独立夹紧到 `[minAngle, maxAngle]`。

**优点**: 支持非对称限制（minY ≠ maxY）
**缺点**: 角落处限制范围不够平滑

### 4.3 特殊情况优化

约束部件通过 flag 位快速判断各轴状态：

| Flag | 含义 | 条件 |
|------|------|------|
| `TwistXLocked` | 扭转锁死 | angle < 0.5° |
| `TwistXFree` | 扭转自由 | angle > 179.5° |
| `SwingYLocked` | Y摆动锁死 | 如肘部 |
| `SwingYFree` | Y摆动自由 | |
| `SwingZLocked` | Z摆动锁死 | |
| `SwingZFree` | Z摆动自由 | |

```
// 计算目标朝向与当前朝向的差异
```

## 六、约束求解流程

每个物理步的求解过程：

```
1. SetupVelocityConstraint(dt)
   ├─ 计算 PointConstraintPart 属性（位置锁定）
   ├─ 计算当前 swing/twist 旋转
   ├─ 调用 SwingTwistConstraintPart::CalculateConstraintProperties()
   │   ├─ 分解 q → q_swing, q_twist
   │   ├─ 夹紧到限制范围
   │   └─ 设置约束 Jacobian + bias
   └─ 设置电机/摩擦约束部件

2. WarmStartVelocityConstraint(ratio)
   ├─ 对所有约束部件应用上一帧的累积冲量 × 比率

3. SolveVelocityConstraint(dt)  [在PGS迭代中多次调用]
   ├─ 求解 PointConstraintPart（位置）
   ├─ 求解 SwingTwistConstraintPart（角度限制）
   ├─ 求解 MotorConstraintPart[0..2]（电机/摩擦）
   │   └─ 电机扭矩夹紧到 MotorSettings 的限制内

4. SolvePositionConstraint(dt, baumgarte)
   ├─ Baumgarte 位置修正
```

## 七、与其他约束类型的对比

| 约束类型 | 自由度 | 适用关节 | 布娃娃推荐度 |
|----------|--------|----------|-------------|
| **SwingTwistConstraint** | 位置锁定 + 摆动/扭转限制 + 电机 | 肩、髋、头、脊柱 | ★★★★★ |
| HingeConstraint | 1轴旋转 | 肘、膝 | ★★★★ |
| ConeConstraint | 锥形限制(无扭转) | 简化关节 | ★★★ |
| PointConstraint | 仅位置锁定 | 调试用 | ★★ |
| FixedConstraint | 完全固定 | 调试用 | ★ |

在 Jolt 的示例中，所有关节统一使用 SwingTwistConstraint，但通过设置 `normalAngle=0°` 来模拟铰链效果（如膝盖只能弯曲不能侧向）。

## 三、三种姿态驱动模式

### 3.1 瞬间设置 — SetPose

```cpp
void Ragdoll::SetPose(const SkeletonPose& inPose);
void Ragdoll::SetPose(RVec3Arg inRootOffset, const Mat44* inJointMatrices);
```

**机制**: 直接调用 `BodyInterface::SetPositionAndRotation()` 瞬间传送每个刚体。

**用途**: 初始化布娃娃位置、重置姿态。

```cpp
    const Mat44& joint = inJointMatrices[i];

```

### 3.2 运动学驱动 — DriveToPoseUsingKinematics

```cpp
void Ragdoll::DriveToPoseUsingKinematics(const SkeletonPose& inPose, float inDeltaTime);
```

**机制**: 调用 `BodyInterface::MoveKinematic()`，计算所需速度使刚体在 `deltaTime` 内到达目标位置。

**用途**: 运动学驱动的布娃娃（motionType = Kinematic），能推动场景中的其他物体。

```cpp
// 内部实现

        mBodyIDs[i],
        rootOffset + joint.GetTranslation(),
        joint.GetQuaternion(),

```

**使用场景示例** (KinematicRigTest):
```cpp
void PrePhysicsUpdate(float dt) {
    // 1. 采样前一帧的姿态（验证布娃娃是否跟上了）
    animation->Sample(prevTime, pose);
    
    // 2. 采样下一帧的目标姿态
    time += dt;
    animation->Sample(time, pose);

}
```

### 3.3 电机驱动 — DriveToPoseUsingMotors

```cpp
void Ragdoll::DriveToPoseUsingMotors(const SkeletonPose& inPose);
```

**机制**: 为每个 SwingTwistConstraint 设置电机目标朝向，让物理引擎通过弹簧力驱动关节到目标旋转。

**用途**: powered ragdoll，布娃娃尝试跟随动画但仍受物理影响（可被推倒、摔倒）。

```cpp
// 内部实现
for (int i = 0; i < jointCount; ++i) {
    int constraintIdx = mRagdollSettings->GetConstraintIndexForBodyIndex(i);
    if (constraintIdx >= 0) {
        const JointState& jointState = inPose.GetJoint(i);
        TwoBodyConstraint* constraint = mConstraints[constraintIdx];
        
        if (constraint->GetSubType() == EConstraintSubType::SwingTwist) {
            SwingTwistConstraint* st = static_cast<SwingTwistConstraint*>(constraint);
            st->SetSwingMotorState(EMotorState::Position);
            st->SetTwistMotorState(EMotorState::Position);
            st->SetTargetOrientationBS(jointState.mRotation);
            // BS = Body Space, 使用局部空间旋转作为目标
        }
    }
}
```

**使用场景示例** (PoweredRigTest):
```cpp
void PrePhysicsUpdate(float dt) {
    time += dt;
    animation->Sample(time, pose);
    
    // 根关节位置从物理世界读取（物理引擎控制）
    Vec3 rootPos;
    Quat rootRot;
    ragdoll->GetRootTransform(rootPos, rootRot);
    pose.GetJoint(0).mTranslation = Vec3::sZero();
    pose.GetJoint(0).mRotation = rootRot;  // 保持物理旋转
    pose.SetRootOffset(rootPos);
    
    pose.CalculateJointMatrices();
    ragdoll->DriveToPoseUsingMotors(pose);
}
```

## 一、SkeletonMapper — 骨骼映射

**源文件**: `Jolt/Skeleton/SkeletonMapper.h/.cpp`

### 1.1 应用场景

游戏中通常有：
- **低精度骨骼**（布娃娃用，~12 关节）
- **高精度骨骼**（动画/渲染用，~60+ 关节）

SkeletonMapper 负责在两者之间转换姿态。

### 1.2 映射类型

| **Unmapped** | 只存在于高精度骨骼中的关节 | 关节索引 + 父索引 |
| **Locked** | 需锁定平移的关节 | 关节索引 + 参考平移 |

### 1.3 初始化

```cpp
SkeletonMapper mapper;
mapper.Initialize(

    animSkeleton,       // 高精度骨骼（skeleton 2）
    animNeutralPose,    // 高精度中性姿态
    canMapJointFunc     // 判断两个关节是否匹配（默认按名称）
);
```

### 1.4 映射方向

```cpp
// 布娃娃 → 动画骨骼（用于渲染）
mapper.Map(ragdollPoseModel, animPoseLocal, animPoseModelOut);

// 动画骨骼 → 布娃娃（用于驱动）
mapper.MapReverse(animPoseModel, ragdollPoseModelOut);
```

### 1.5 平移锁定

约束永远不是100%刚性的，物理仿真中布娃娃会有微小拉伸。`LockTranslations()` 或 `LockAllTranslations()` 可以锁定高精度骨骼关节的平移，消除视觉拉伸（代价是物理与视觉有微小偏差）。

## 二、仿写指南 — 在自己的物理后端实现布娃娃

### 2.1 分层实现计划

建议按以下顺序递进实现：

```
Level 0: 前置条件（你的物理后端需要有的）
  ├─ 刚体系统 (Body + 速度积分 + 碰撞)  ← 已有
  ├─ 约束系统框架 (Constraint + 求解)     ← 需要确认
  └─ PointConstraint (锁定两点位置)      ← 基础约束

Level 1: 基础布娃娃
  ├─ Skeleton (关节树)
  ├─ SkeletonPose (姿态)
  ├─ 简化版 SwingTwist 约束或 BallJoint 约束
  ├─ RagdollSettings + Ragdoll
  └─ SetPose() 初始化

Level 2: 碰撞过滤
  └─ 父子碰撞禁用 (GroupFilterTable 或简单的碰撞对排除)

Level 3: 稳定化
  └─ Stabilize() 质量/惯性平衡

Level 4: 驱动模式
  ├─ DriveToPoseUsingKinematics (运动学)
  └─ DriveToPoseUsingMotors (电机，需要约束电机支持)

Level 5: 高级功能（可选）
  ├─ SkeletonMapper
  └─ 动画混合
```

### 2.2 最小可行布娃娃的核心代码量

| 模块 | 估计代码量 | 说明 |
|------|-----------|------|
| Skeleton + SkeletonPose | ~100行 | 纯数据结构 + CalculateJointMatrices |
| RagdollSettings + Ragdoll | ~200行 | 管理容器，核心是创建/销毁逻辑 |
| SwingTwist约束 | ~300-500行 | **最有挑战的部分** |
| 碰撞过滤 | ~50行 | 简单的排除对列表 |
| Stabilize | ~100行 | 质量重分配算法 |
| **合计** | ~750-950行 | |

### 2.3 SwingTwist 约束的简化实现思路

完整的 SwingTwistConstraintPart 比较复杂，可以分步简化：

**Phase 1: BallJoint（最简单）**
```
只实现 PointConstraintPart：
- 锁定两个锚点位置（3-DOF 位置约束）
- 不限制旋转
- 布娃娃可以自由旋转但不会散架
```

**Phase 2: BallJoint + 锥形限制**
```
在 BallJoint 基础上增加：
- 分解相对旋转为 swing + twist
- 限制 twist 角度: clamp(twist_angle, min, max)
- 限制 swing 角度: 如果超出锥形范围则投影回去
- 用 AngleConstraintPart 求解各轴的角度约束
```

**Phase 3: 加入电机**
```
- 计算目标朝向与当前朝向的差异
- 用弹簧约束驱动差异收敛到零
```

### 2.4 关键数学速查

**四元数 Swing-Twist 分解**:
```
输入: q (相对旋转四元数)
输出: q_swing, q_twist

// twist 只包含绕 X 轴的旋转
q_twist = normalize(Quat(q.x, 0, 0, q.w))
// 如果 twist 接近零则设为 identity
if |q_twist| ≈ 0: q_twist = Quat(0, 0, 0, 1)

// swing = q * twist^-1
q_swing = q * conjugate(q_twist)
```

**锥形限制（椭圆投影）**:
```
point = (q_swing.y, q_swing.z)
ellipse: (y/a)² + (z/b)² ≤ 1
  where a = sin(maxSwingY/2), b = sin(maxSwingZ/2)

if point outside ellipse:
    closest = 找到椭圆上最近点 (解析解或牛顿迭代)
    q_swing = Quat(0, closest.x, closest.y, sqrt(1 - closest.x² - closest.y²))
```

**角度约束求解（单轴）**:
```
// 与 AxisConstraintPart 类似
axis = 约束轴 (世界空间)
angle_error = current_angle - target_angle

// 有效质量 (类似碰撞约束)
K = (r1×axis)^T * I1_inv * (r1×axis) + (r2×axis)^T * I2_inv * (r2×axis)
effectiveMass = 1 / K

// 求解冲量
lambda = -effectiveMass * (J * v + bias)

// 施加冲量
body1.angularVelocity += I1_inv * (r1 × axis) * lambda
body2.angularVelocity -= I2_inv * (r2 × axis) * lambda
```

### 2.5 容易踩的坑

1. **四元数符号**: 四元数 q 和 -q 代表同一旋转，分解 swing/twist 前必须确保 w > 0
2. **万向锁**: SwingTwist 分解在某些极端角度下不稳定，Jolt 用 flag 处理锁死/自由的特殊情况
3. **约束爆炸**: 不调 Stabilize 的布娃娃在受力时会剧烈抖动
4. **自碰撞**: 不禁用父子碰撞会导致初始化时肢体互相弹飞
5. **warm start**: ResetWarmStart 在 SetPose 后调用，否则上一帧的约束冲量会造成跳变
6. **约束位置**: 约束锚点应该在两个肢体的交界处（关节位置），不在肢体中心
7. **质量分布**: 远端肢体（手、脚）不应比近端肢体（大腿、上臂）更重

## 四、总结：Jolt 布娃娃的设计哲学

1. **布娃娃 = 刚体 + 约束的组织管理层**，不引入新的物理概念
2. **配置与运行时分离**: RagdollSettings（模板，可复用） → Ragdoll（实例）
3. **一切关节统一用 SwingTwistConstraint**，通过参数退化为铰链/锥形
4. **稳定化是必须的**，不做质量/惯性平衡会导致约束爆炸
5. **碰撞过滤是必须的**，父子肢体在初始状态通常重叠
6. **三种驱动模式覆盖所有场景**：瞬间定位 / 运动学推动 / 物理驱动

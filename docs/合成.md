## 四、完整物理更新流水线（单步简化顺序）

```
Step 0: 前处理 — 清零 force/torque，调用 StepListener
Step 1: 速度积分 — 重力+外力→速度，阻尼，陀螺力（可选）
Step 2: 宽相位碰撞检测 — AABB重叠对收集
Step 3: 窄相位碰撞检测 — GJK求距离，EPA求穿透
Step 4: 接触流形生成 — ClipPoly裁剪，PruneContactPoints精简到≤4点
Step 5: 约束组装 — 每个接触点创建 法线约束+2个切线摩擦约束
Step 6: Warm Start — 用上帧λ乘(dt_new/dt_old)缩放后施加
Step 7: 速度约束求解 — 迭代10次(默认)，每次先摩擦后法线
Step 8: 位置积分 — p += v*dt, q = Quat(ω*dt) * q
Step 9: 位置约束修正 — Baumgarte(β=0.2)，迭代2次(默认)
Step 10: 休眠检测与回调
```

> 后续文档 01~09 将逐一展开每个步骤的数学细节与实现要点。
# Jolt 刚体物理核心算法 — 数据结构定义

## 一、刚体 (Body) 的最小必要字段

从 Jolt 的 `Body.h` 和 `MotionProperties.h` 中提取，仿写只需要以下字段：

```c
struct RigidBody {
    // ======== 几何状态 ========
    Vec3   position;          // 质心世界位置 (m)
    Quat   rotation;          // 朝向四元数 (单位四元数)
    Shape* shape;             // 碰撞形状引用 (球/盒/凸体)

    // ======== 运动状态 ========
    Vec3   linearVelocity;    // 质心线速度 (m/s)
    Vec3   angularVelocity;   // 角速度 (rad/s)，世界空间

    // ======== 质量属性 (仅 Dynamic 需要) ========
    float  invMass;           // 逆质量 = 1/m (1/kg)
    Vec3   invInertiaDiag;    // 逆惯性张量对角线 D = (dx, dy, dz)
    Quat   inertiaRotation;   // 惯性主轴旋转 R
    // 关系: I_body^{-1} = R * diag(D) * R^{-1}

    // ======== 力/扭矩累积 (每帧清零) ========
    Vec3   force;             // 累积力 (N)
    Vec3   torque;            // 累积扭矩 (N·m)

    // ======== 材质属性 ========
    float  friction;          // 摩擦系数 (无量纲, 通常 0~1)
    float  restitution;       // 恢复系数 (0=完全非弹性, 1=完全弹性)

    // ======== 阻尼 ========
    float  linearDamping;     // 线性阻尼系数 c (v *= max(0, 1-c*dt))
    float  angularDamping;    // 角阻尼系数
    float  gravityFactor;     // 重力缩放因子 (1.0=正常, 0=无重力)

    // ======== 类型 ========
    enum MotionType {
        Static,     // 不可移动 (invMass=0, 无MotionProperties)
        Kinematic,  // 按速度移动，但不受力影响 (invMass视为0)
        Dynamic     // 受力+碰撞完全影响
    } motionType;
};
```

## 二、惯性张量的对角化存储

Jolt **不存储完整 3×3 逆惯性矩阵**。在创建刚体时，通过
`MassProperties::DecomposePrincipalMomentsOfInertia()` 做特征值分解：

```
I_body^{-1} = R · diag(dx, dy, dz) · R^{-1}
```

存储为:
- `invInertiaDiag` = (dx, dy, dz)  — 3个float
- `inertiaRotation` = R            — 1个Quat (4个float)

**对比完整矩阵存储 (9个float)**，节省内存且计算更快。

### 世界空间逆惯性乘向量

这是最频繁的操作，Jolt 在 `MotionProperties.inl` 中实现：

```
MultiplyWorldSpaceInverseInertiaByVector(bodyRotation, v):
    // 组合旋转: body朝向 * 惯性主轴旋转
    combinedRot = bodyRotation * inertiaRotation
    rotMatrix = Mat44::sRotation(combinedRot)

    // 转到惯性主轴空间 → 逐分量乘对角线 → 转回世界空间
    localV = rotMatrix^T * v          // 世界→本体主轴
    scaled = invInertiaDiag ⊙ localV  // 逐分量乘
    result = rotMatrix * scaled       // 本体主轴→世界
    return result
```

### 常见形状的惯性张量

| 形状 | 惯性张量 I (质量m) | 说明 |
|------|-------------------|------|
| 实心球 (半径r) | (2/5)mr² · I₃ | 对角线全相等 |
| 实心长方体 (a×b×c) | m/12 · diag(b²+c², a²+c², a²+b²) | 对角化已完成 |
| 实心圆柱 (半径r, 高h) | diag(m(3r²+h²)/12, m(3r²+h²)/12, mr²/2) | 绕轴对称 |

对角化后 `inertiaRotation` 对于上述对称形状就是单位四元数。
只有不对称凸体才会有非平凡的旋转。

## 三、接触点数据结构

```c
struct ContactPoint {
    Vec3  position1;   // body1 表面上的接触点 (质心相对坐标)
    Vec3  position2;   // body2 表面上的接触点 (质心相对坐标)
    float penetration; // 穿透深度 (>0 表示穿透, <0 表示分离)
};

struct ContactManifold {
    Body* body1;
    Body* body2;
    Vec3  normal;       // 碰撞法线 (世界空间, body1→body2 方向)
    Vec3  tangent1;     // 切线方向1 (与 normal 正交)
    Vec3  tangent2;     // 切线方向2 (与 normal、tangent1 正交)
    float combinedFriction;
    float combinedRestitution;
    ContactPoint points[4];  // 最多4个接触点
    int   numPoints;
};
```

Jolt 的做法：摩擦和恢复系数默认取两体的**几何平均值**:
- `combinedFriction = sqrt(body1.friction * body2.friction)`
- `combinedRestitution = max(body1.restitution, body2.restitution)`

这些可以在 `PhysicsSystem::SetCombineFriction/Restitution` 中自定义。

## 四、碰撞形状的 Support 函数接口

GJK/EPA 不关心具体形状，只需要一个 `GetSupport(direction)` 函数：

```c
// 返回形状在 direction 方向上最远的点
Vec3 GetSupport(Vec3 direction);
```

常见形状的 Support 函数:

| 形状 | GetSupport(d) |
|------|--------------|
| 球 (中心c, 半径r) | `c + r * normalize(d)` |
| 长方体 (半尺寸h) | `(sign(d.x)*h.x, sign(d.y)*h.y, sign(d.z)*h.z)` |
| 凸多面体 (顶点集V) | `argmax_{v∈V} (v · d)` |
| 胶囊体 | 两球心取更远的 + r * normalize(d) |

Jolt 在 `ConvexSupport.h` 中还提供了组合包装:
- **`MinkowskiDifference<A,B>`**: `A.GetSupport(d) - B.GetSupport(-d)`
- **`AddConvexRadius<A>`**: `A.GetSupport(d) + radius * normalize(d)`
  （在 GJK 核心中不含 convex radius，EPA 中才包含）
# Jolt 刚体物理核心算法 — 速度积分与力

对应 Jolt 源码: `MotionProperties.inl` 中的
`ApplyForceTorqueAndDragInternal` 和 `ApplyGyroscopicForceInternal`，
以及 `Body.inl` 中的 `AddRotationStep`。

## 一、力→速度积分 (半隐式欧拉)

在每个物理步的**开头**，对所有 Dynamic body 执行：

```
输入: body, gravity, dt
前提: body.force / body.torque 已在此帧内通过 AddForce/AddTorque 累积

// ---- 线速度更新 ----
v += dt * (gravityFactor * gravity + invMass * force)

// ---- 角速度更新 ----
// 先将扭矩通过世界空间逆惯性转为角加速度
angularAccel = I_world_inv * torque   // 见01文档中的 MultiplyWorldSpaceInverseInertiaByVector
ω += dt * angularAccel

// ---- 阻尼 ----
// 精确解: v(t) = v0 * e^(-c*t)
// 一阶泰勒近似: v *= max(0, 1 - c * dt)
v *= max(0.0, 1.0 - linearDamping * dt)
ω *= max(0.0, 1.0 - angularDamping * dt)

// ---- 速度钳制 ----
if |v|² > maxLinearVelocity²:
    v *= maxLinearVelocity / |v|
if |ω|² > maxAngularVelocity²:
    ω *= maxAngularVelocity / |ω|

// ---- 清零累积力 ----
force = (0, 0, 0)
torque = (0, 0, 0)
```

**要点**:
- 这是**半隐式欧拉 (Symplectic Euler)**：先更新速度，再用新速度更新位置
- 阻尼的 `max(0, ...)` 保护是为了避免 dt 极大或 damping 极大时产生负缩放
- Jolt 默认 `maxLinearVelocity = 500 m/s`，`maxAngularVelocity = 0.25π * 60 rad/s`

## 二、陀螺力 (Gyroscopic Force)

**可选功能**，仅对标记了 `ApplyGyroscopicForce` 的body启用。
对应物理现象: **Dzhanibekov 效应** (网球拍定理)。

```
输入: body.rotation, body.angularVelocity (ω), dt

// 1. 求本体惯性 (对角形式的逆运算)
// invInertiaDiag = (dx, dy, dz)
// localInertia = (1/dx, 1/dy, 1/dz)  (某分量为0则结果也为0)
localInertia.x = (dx != 0) ? 1.0/dx : 0.0
localInertia.y = (dy != 0) ? 1.0/dy : 0.0
localInertia.z = (dz != 0) ? 1.0/dz : 0.0

// 2. 转到惯性主轴空间
Q = bodyRotation * inertiaRotation   // 世界→惯性主轴 的组合旋转
ω_local = Q^{-1} * ω                // 角速度转到主轴空间

// 3. 计算本体角动量
L = localInertia ⊙ ω_local          // 逐分量相乘

// 4. 施加陀螺扭矩: T = -ω × (I·ω)
// 用显式欧拉更新角动量
L_new = L - dt * (ω_local × L)

// 5. 关键: 长度守恒 — 防止欧拉积分注入能量
// 将新角动量缩放到和旧角动量相同的长度
if |L_new|² > 0:
    L_new *= sqrt(|L|² / |L_new|²)
else:
    L_new = (0,0,0)

// 6. 转回世界空间角速度
ω = Q * (invInertiaDiag ⊙ L_new)
```

**核心技巧**: 第5步的长度归一化。如果不做这一步，由于欧拉积分的
数值误差，角动量会逐渐增大导致物体越转越快（能量不守恒）。

## 三、位置积分 (在约束求解之后)

位置积分发生在**速度约束求解之后**（此时速度已被约束修正）：

### 3.1 平移积分
```
position += linearVelocity * dt
```
极其简单，直接欧拉积分。

### 3.2 旋转积分

Jolt **不使用**一阶近似 `q' = q + 0.5*dt*(0,ω)*q`，而是用**精确轴角**：

```
// 来自 Body.inl 的 AddRotationStep
angVelDt = angularVelocity * dt    // 角速度×时间 = 旋转向量
angle = |angVelDt|                 // 旋转角度 (rad)
if angle > 1e-6:
    axis = angVelDt / angle        // 旋转轴 (归一化)
    deltaQ = Quat::FromAxisAngle(axis, angle)  // 增量四元数
    rotation = Normalize(deltaQ * rotation)    // 左乘并归一化
```

**为什么不用一阶近似?**
- 一阶近似 `q += 0.5*dt*ω*q` 在大角速度时误差很大
- 对动画驱动的 Kinematic ragdoll，帧间旋转差异可能很大
- 轴角方法在任意角速度下都精确，且自动保持四元数单位长度

### 3.3 Sub步骤的反向旋转

`SubRotationStep` 用于位置修正阶段"回退"旋转：
```
// 与 AddRotationStep 完全相同，只是角度取反
deltaQ = Quat::FromAxisAngle(axis, -angle)
rotation = Normalize(deltaQ * rotation)
```

### 4.1 线冲量 (质心)

### 4.2 线+角冲量 (任意点)

# Jolt 刚体物理核心算法 — GJK 碰撞检测

对应 Jolt 源码: `Geometry/GJKClosestPoint.h` 和 `Geometry/ClosestPoint.h`。
基于 Gino van den Bergen 的方法。

## 一、GJK 算法概述

GJK (Gilbert-Johnson-Keerthi) 求解**两个凸体之间的最短距离**。

核心思想：在 Minkowski 差 `A ⊖ B = {a-b | a∈A, b∈B}` 上找**离原点最近的点**。
- 如果最近点距离 > 0 → 不相交，距离就是两体间距
- 如果原点在 Minkowski 差内部 → 相交（交给 EPA 处理）

## 二、Support 函数

GJK 只需要每个形状提供一个 Support 函数：

```
Support_A(d) = argmax_{a∈A} (a · d)   // A中在d方向最远的点
Support_B(d) = argmax_{b∈B} (b · d)

// Minkowski差的Support:
Support_{A⊖B}(d) = Support_A(d) - Support_B(-d)
```

## 三、GJK 核心循环 (GetClosestPoints)

```
输入: 凸体A, 凸体B, tolerance, maxDistSq
输出: 最近点 pointA (在A上), pointB (在B上), 距离²

simplex Y = {}          // 单纯形点集 (最多4个点)
P[], Q[] = {}           // 对应A上/B上的支撑点
v = 初始分离轴           // 任意非零向量
v_len_sq = |v|²
prev_v_len_sq = FLT_MAX

loop:
    // 1. 取支撑点
    p = A.GetSupport(v)       // A在v方向最远
    q = B.GetSupport(-v)      // B在-v方向最远
    w = p - q                 // Minkowski差上的新点

    // 2. 分离检测: 如果w在v方向的投影为负，距离太远
    dot = v · w
    if dot < 0 且 dot² > v_len_sq * maxDistSq:
        return FLT_MAX  // 距离超过阈值，提前返回

    // 3. 加入单纯形
    Y.add(w); P.add(p); Q.add(q)

    // 4. 求单纯形上离原点最近的点
    // 调用 ClosestPoint 子系统（见下文）
    (v_new, set) = GetClosestPointOnSimplex(Y)

    // 5. 判断终止条件
    if set == 0xF:           // 4个点都保留 = 原点在四面体内部
        v = (0,0,0)          // 碰撞! 距离=0
        break

    if |v_new|² ≤ tolerance²:  // 足够近
        v = (0,0,0); break

    if |v_new|² ≤ FLT_EPSILON * max(|Y_i|²):  // 机器精度
        v = (0,0,0); break

    // 6. 收敛检测: 距离不再减小
    v = -v_new               // 下一个搜索方向 = 反向最近点
    if prev_v_len_sq - |v_new|² ≤ FLT_EPSILON * prev_v_len_sq:
        break                // 已收敛

    prev_v_len_sq = |v_new|²

    // 7. 精简单纯形: 只保留 set 中的点
    RemovePointsNotInSet(Y, P, Q, set)

// 8. 用重心坐标插值出A/B上的最近点
pointA, pointB = CalculateFromBarycentricCoords(Y, P, Q)
return |v|²
```

## 四、ClosestPoint 子系统

`ClosestPoint.h` 提供离原点最近点的计算，分4种情况:

### 4.1 线段上最近点 (2点)
```
GetClosestPointOnLine(A, B):
    AB = B - A
    denom = |AB|²
    if denom < ε²:  // 退化为点
        return 距原点更近的那个
    v = -A·AB / denom    // B的重心坐标
    u = 1 - v
    if v ≤ 0: return A (set=01)
    if u ≤ 0: return B (set=10)
    return u*A + v*B (set=11)
```

### 4.2 三角形上最近点 (3点)

使用 Christer Ericson 的 "Real-Time Collision Detection" 算法:

```
GetClosestPointOnTriangle(A, B, C):
    // 1. 选择最短边参与计算以提高精度
    // 2. 计算三角形法线 n = 最短边 × 另一边
    // 3. 测试原点在各边的哪一侧
    // 4. 分情况返回:
    //    - 原点投影在三角形内部 → 返回投影点
    //    - 原点最近某条边 → 退化到线段
    //    - 原点最近某个顶点 → 直接返回顶点
    
    // 重心坐标 (u,v,w) 满足 u+v+w=1
    // 使用行列式公式计算:
    d00 = v0·v0;  d01 = v0·v1;  d11 = v1·v1
    denom = d00*d11 - d01*d01  // ≥ 0 (柯西不等式)
    if denom < 1e-10: // 退化三角形，退到最长边
        ...
    返回 set 位掩码标识哪些顶点参与
```

### 4.3 四面体上最近点 (4点)

```
GetClosestPointOnTetrahedron(A, B, C, D):
    // 1. 检查原点是否在四面体内部
    //    通过测试原点在每个面的哪一侧
    // 2. 如果在内部: set = 0xF, 返回原点本身
    // 3. 如果在外部: 测试4个面，找最近的面
    //    对最近的面退化到三角形情况
```

### 4.4 GJK的模板优化

Jolt 有一个关键优化 `LastPointPartOfClosestFeature`:
- 最后加入的点必然在最近特征中（因为它是沿着最近方向取的支撑点）
- 这允许跳过很多不必要的子特征测试

## 五、Convex Radius (膨胀半径)

Jolt对所有凸体形状都有一个 `ConvexRadius` 概念:
- GJK 核心运算使用**不含radius的形状** (ExcludingConvexRadius)
- 求出距离后再加上两体的 radius 之和
- 如果 `distance ≤ radiusA + radiusB` → 碰撞

这样做的好处: 
- GJK 收敛更快 (不含radius的Minkowski差更"尖锐")
- 圆角效果自动获得
- EPA 则使用**含radius的形状** (IncludingConvexRadius)

```
// EPAPenetrationDepth.h: GetPenetrationDepthStepGJK
combined_radius = radiusA + radiusB
dist_sq = GJK.GetClosestPoints(A_excl, B_excl, ...)

if dist_sq > combined_radius²:
    return NotColliding

if dist_sq > 0:
    // 在convex radius内碰撞
    // 调整接触点: 沿分离方向各偏移自己的radius
    pointA += direction * (radiusA / dist)
    pointB -= direction * (radiusB / dist)
    return Colliding

// dist_sq == 0: 深度穿透，需要EPA
return Indeterminate
```
# Jolt 刚体物理核心算法 — EPA 穿透深度计算

对应 Jolt 源码: `Geometry/EPAPenetrationDepth.h` 和 `Geometry/EPAConvexHullBuilder.h`。
基于 Gino van den Bergen 的 Expanding Polytope Algorithm。

## 一、EPA 的作用

当 GJK 判定两凸体**相交** (原点在 Minkowski 差内部) 后，
GJK 只能告诉你"相交了"，但无法给出：
- **穿透深度** (penetration depth)
- **穿透方向** (分离法线)
- **最近接触点对**

EPA 就是解决这个问题的。

## 二、EPA 算法概述

```
输入: GJK的最终单纯形 (包含原点的3~4个点), 两凸体的Support函数
输出: 穿透法线, 穿透深度, A/B上的接触点

思路:
  把GJK的单纯形扩展成一个凸包 (polytope)
  不断找凸包上离原点最近的面
  用该面的法线方向取新的支撑点来扩展凸包
  直到新点不再显著靠近原点 → 收敛
```

## 三、详细算法流程

### Step 1: 初始化凸包

```
// 从GJK得到的单纯形可能是1~4个点
// 需要保证至少4个点形成非退化四面体

if numPoints < 4:
    // 补充支撑点: 沿坐标轴方向取新支撑点
    // 直到有4个不共面的点
    补点方向 = ±x, ±y, ±z 依次尝试

// 用这4+个点构建初始凸包 (三角形面集合)
convexHull = EPAConvexHullBuilder(points)

// 验证: 原点必须在凸包内部
// 如果不在 → GJK判断有误或数值问题 → 返回false
```

### Step 2: 扩展循环

```
loop:
    // 1. 从所有三角形面中找离原点最近的面
    closestTriangle = 凸包面中 (面到原点距离²) 最小的
    closestDistSq = 该面到原点的距离²
    normal = 该面的外法线

    // 2. 沿法线方向取新支撑点
    w = Support_{A⊖B}(normal)   // Minkowski差的新支撑点
    // 同时记录 p=A.Support(normal), q=B.Support(-normal)

    // 3. 收敛检测
    newDistSq = (w · normal)² / |normal|²
    if newDistSq - closestDistSq < tolerance * closestDistSq:
        break  // 新点没有显著改善，收敛了

    // 4. 将新点加入凸包
    // - 删除所有"面向新点"的三角形 (新点在其正面)
    // - 用新点和暴露的边缘构建新三角形
    // 这就是凸包的增量构建
    convexHull.AddPoint(w, p, q)

// 5. 从最近面的重心坐标插值出A/B上的接触点
// 最近面有3个顶点Y[i], 对应P[i]和Q[i]
// 用重心坐标 (u,v,w) 插值:
pointA = u*P[0] + v*P[1] + w*P[2]
pointB = u*Q[0] + v*Q[1] + w*Q[2]

// 穿透法线 = normalize(pointA - pointB) 或 closestTriangle法线方向
// 穿透深度 = |pointA - pointB|
```

## 四、EPAConvexHullBuilder

Jolt 不使用简单的"三角形分裂"方法，而是真正的增量凸包构建:

```
数据结构:
    Triangle {
        int indices[3];       // 顶点索引
        Vec3 normal;          // 外法线
        float closestDistSq;  // 到原点距离²
        Triangle* adjacent[3]; // 邻接三角形
    }
    优先队列 (按closestDistSq排序)

AddPoint(newPoint):
    1. 标记所有面向新点的三角形 (newPoint在其法线正侧)
    2. 收集这些三角形的外边界 (horizon edges)
       — 即: 一侧三角形被删除、另一侧保留的边
    3. 删除所有被标记的三角形
    4. 用新点 + 每条horizon edge 生成新三角形
    5. 设置邻接关系
    6. 加入优先队列
```

**这种方法的优势**: 避免产生极细长的三角形 (文章中提到的问题)，
保持凸包质量良好。

## 五、两步法: GJK + EPA 的完整流程

Jolt 在 `EPAPenetrationDepth.h` 中将两者串联:

```
// Step 1: GJK
status = GetPenetrationDepthStepGJK(
    A_excludingRadius, radiusA,
    B_excludingRadius, radiusB,
    tolerance, ioV, outPointA, outPointB)

switch status:
    case NotColliding:
        return false  // 无碰撞

    case Colliding:
        // GJK已找到接触点 (在convex radius范围内)
        // outPointA/B 已调整了 radius
        return true

    case Indeterminate:
        // 深度穿透，需要EPA
        // Step 2: EPA
        return GetPenetrationDepthStepEPA(
            A_includingRadius,  // 注意这里用含radius版本
            B_includingRadius,
            tolerance, outV, outPointA, outPointB)
```

## 六、简化版仿写要点

如果只处理简单形状 (球、盒)，可以大幅简化:

| 形状对 | GJK需要？ | EPA需要？ | 替代方案 |
|--------|----------|----------|---------|
| 球 vs 球 | 不需要 | 不需要 | 直接解析: dist = |c1-c2| - r1 - r2 |
| 球 vs 盒 | 不需要 | 不需要 | 最近点投影公式 |
| 盒 vs 盒 | 可以SAT替代 | 可以SAT替代 | SAT (15轴) 更直觉 |
| 凸体 vs 凸体 | **需要** | **需要** | 无替代 |

对于完整凸体，GJK+EPA 是唯一实用的通用算法。
# Jolt 刚体物理核心算法 — 接触流形生成

对应 Jolt 源码: `Geometry/ClipPoly.h` 和
`Physics/Collision/ManifoldBetweenTwoFaces.cpp`。

## 一、为什么需要接触流形？

GJK/EPA 只返回**一个接触点对**。但物理模拟需要**多个接触点**:
- 一个盒子平放在地面上，需要4个接触点才能稳定
- 只有1个点会导致物体不断"晃动"

**接触流形 (Contact Manifold)** 就是从碰撞信息中生成最多4个
稳定接触点的过程。

## 二、整体流程

```
GJK/EPA 输出:
    contactPoint1   (body1上的接触点)
    contactPoint2   (body2上的接触点)  
    penetrationAxis (穿透方向/碰撞法线)

流程:
    1. 获取两体各自的 Supporting Face (朝碰撞法线方向的面)
    2. Clip face2 against face1 (多边形裁剪)
    3. 将裁剪结果投影到 face1 平面上
    4. 如果接触点 > 4，PruneContactPoints 裁剪到 ≤ 4
```

## 三、Supporting Face (支撑面)

每个碰撞形状除了 `GetSupport(direction)` 外，
还需提供 `GetSupportingFace(direction)`:

```
// 返回朝 direction 方向最对齐的面的顶点列表
void GetSupportingFace(Vec3 direction, vertices[]);
```

| 形状 | 支撑面 |
|------|--------|
| 球 | 空（永远退化为点接触） |
| 盒 | direction最大分量对应的面 (4个顶点) |
| 凸包 | 法线最接近direction的面 |
| 胶囊 | 如果direction接近轴向→端面圆(边)，否则→空 |

## 四、ClipPolyVsPoly — 多边形裁剪

核心算法: **Sutherland-Hodgman** 逐边裁剪。

```
// ClipPoly.h: ClipPolyVsPoly
输入:
    polygonToClip   — shape2的面 (被裁剪方)
    clippingPolygon — shape1的面 (裁剪方, ≥3顶点)
    clippingNormal  — shape1面法线

算法:
    对 clippingPolygon 的每条边:
        // 计算该边的内法线
        edge = clippingPolygon[(i+1) % n] - clippingPolygon[i]
        clipNormal = clippingNormal × edge  // 指向clippingPolygon内部

        // 用该半平面裁剪当前多边形
        ClipPolyVsPlane(currentPoly, edgeVertex, clipNormal, resultPoly)

        // 如果结果 < 3个顶点，没有交集了
        if resultPoly.size() < 3: return empty

    return resultPoly  // 裁剪后的多边形
```

### ClipPolyVsPlane — 逐边裁剪的单步

```
// 逐顶点遍历多边形，对每条边判断:
for each edge (e1 → e2):
    e1_inside = (planeOrigin - e1) · planeNormal < 0
    e2_inside = (planeOrigin - e2) · planeNormal < 0

    if e1_inside != e2_inside:
        // 一进一出: 计算边与平面的交点
        t = (planeOrigin - e1) · planeNormal / (e2 - e1) · planeNormal
        intersection = e1 + t * (e2 - e1)
        output.add(intersection)

    if e2_inside:
        output.add(e2)   // 内部点保留
```

## 五、ManifoldBetweenTwoFaces — 完整流形生成

```
ManifoldBetweenTwoFaces(point1, point2, axis, maxDist, face1, face2):

    if face1.size ≥ 2 && face2.size ≥ 3:
        // 裁剪 face2 against face1
        if face1.size ≥ 3:
            clipped = ClipPolyVsPoly(face2, face1, axis)
        else if face1.size == 2:
            clipped = ClipPolyVsEdge(face2, face1[0], face1[1], axis)

        // 确定 face1 的平面
        planeOrigin = face1[0]
        planeNormal = (face1[1] - face1[0]) × (face1[2] - face1[0])

        // 将裁剪后的每个点投影到 face1 平面上
        for each p2 in clipped:
            // 沿 penetrationAxis 方向投影
            distance = (p2 - planeOrigin) · planeNormal / (axis · planeNormal)

            // 只保留在阈值内的点
            if distance * |axis| < maxDist:
                p1 = p2 - distance * axis  // 对应的face1上的点
                contacts.add(p1, p2)

    // 如果没有裁剪到任何点，退回到GJK/EPA的单点
    if contacts.empty():
        contacts.add(point1, point2)
```

## 六、PruneContactPoints — 精简到 ≤ 4 点

当裁剪产生 > 4 个接触点时，需要精简。
Jolt 使用**最大面积启发式**:

```
PruneContactPoints(axis, points1, points2):
    // 前提: 接触点 > 4 个

    // 1. 将所有点投影到接触平面上 (去掉法线方向分量)
    projected[i] = points1[i] - (points1[i] · axis) * axis

    // 2. 计算每个点的穿透深度² (作为权重)
    depthSq[i] = |points2[i] - points1[i]|²

    // 3. 选第1个点: 距离质心最远 × 穿透最深 (启发式)
    point1 = argmax_i( |projected[i]|² * depthSq[i] )

    // 4. 选第2个点: 离 point1 最远 × 穿透最深
    point2 = argmax_i( |projected[i] - projected[point1]|² * depthSq[i] )

    // 5. 选第3、4个点: 在 point1-point2 连线两侧各找最远的
    perp = (projected[point2] - projected[point1]) × axis
    point3 = 在 perp 负侧最远的点
    point4 = 在 perp 正侧最远的点

    // 结果: 1~4 个点，构成最大面积四边形
```

**关键设计**: 用 `|距离|² × |穿透深度|²` 作为启发式权重，
确保同时选择"位置分散"和"穿透深"的点，提高稳定性。
# Jolt 刚体物理核心算法 — 约束求解器 (AxisConstraintPart)

对应 Jolt 源码: `Physics/Constraints/ConstraintPart/AxisConstraintPart.h`
和 `Physics/Constraints/ConstraintPart/SpringPart.h`。

参考文献: "Constraints Derivation for Rigid Body Simulation in 3D" - Daniel Chappuis

## 一、约束求解的数学框架

### 核心问题
物理引擎需要满足各种约束 (不穿透、摩擦、关节等)。
每个约束表示为一个方程 `C(x) = 0` 或不等式 `C(x) ≥ 0`。

对速度层面，约束变为:
```
dC/dt = J · v + b = 0
```

其中:
- `J` = 雅可比矩阵 (约束对广义速度的偏导)
- `v` = 广义速度向量 [v1, ω1, v2, ω2]
- `b` = 偏置项 (bias)

### 求解方法: 顺序冲量 (Sequential Impulse)

对每个约束，计算一个冲量 λ 使约束被满足:

```
λ = -K^{-1} (J · v + b)
```

其中 K (有效质量的逆) 为:
```
K = J · M^{-1} · J^T
```

M^{-1} 是广义逆质量矩阵 = diag(1/m1, I1^{-1}, 1/m2, I2^{-1})。

然后用 λ 更新速度:
```
v' = v + M^{-1} · J^T · λ
```

## 二、AxisConstraintPart — 1-DOF 约束

这是 Jolt 中**最核心的约束求解单元**。
碰撞法线约束和摩擦约束都是它的实例。

### 约束方程

沿轴 n (单位向量) 的距离约束:
```
C = (p2 - p1) · n
```

### 雅可比矩阵 J

```
J = [-n^T,  -(r1+u)×n)^T,  n^T,  (r2×n)^T]
```

其中:
- `n` = 约束轴 (归一化)
- `p1, p2` = 约束点 (世界空间)
- `r1 = p1 - x1` (body1质心到约束点)
- `r2 = p2 - x2` (body2质心到约束点)
- `u = x2 + r2 - x1 - r1 = p2 - p1`
- `x1, x2` = body1、body2 的质心

### 有效质量 K

```
K = J · M^{-1} · J^T

展开:
K = 1/m1 + ((r1+u)×n)^T · I1^{-1} · ((r1+u)×n)
  + 1/m2 + (r2×n)^T · I2^{-1} · (r2×n)
```

这是一个标量 (1×1 矩阵)，因为是 1-DOF 约束。

### CalculateConstraintProperties (预计算)

```
CalculateConstraintProperties(body1, r1PlusU, body2, r2, axis, bias):
    // 缓存中间量
    r1PlusU_cross_axis = (r1+u) × axis
    r2_cross_axis      = r2 × axis

    // body1如果是Dynamic:
    invI1_r1PlusU_cross_axis = I1_world_inv * r1PlusU_cross_axis
    K += 1/m1 + invI1_r1PlusU_cross_axis · r1PlusU_cross_axis

    // body2如果是Dynamic:
    invI2_r2_cross_axis = I2_world_inv * r2_cross_axis
    K += 1/m2 + invI2_r2_cross_axis · r2_cross_axis

    effectiveMass = 1.0 / K    // 有效质量
```

### SolveVelocityConstraint (每次迭代调用)

```
SolveVelocityConstraint(body1, body2, axis, minLambda, maxLambda):
    // 1. 计算 J·v (约束违反速度)
    jv = axis · (v1 - v2)                          // 线速度部分
    jv += r1PlusU_cross_axis · ω1                   // body1角速度部分
    jv -= r2_cross_axis · ω2                         // body2角速度部分

    // 2. 计算拉格朗日乘子
    // λ = -K^{-1} (Jv + b)
    // 其中 b 已被 SpringPart 处理 (见下文)
    lambda = effectiveMass * (jv - bias)

    // 3. 累积并钳制
    newTotalLambda = totalLambda + lambda
    newTotalLambda = clamp(newTotalLambda, minLambda, maxLambda)
    deltaLambda = newTotalLambda - totalLambda
    totalLambda = newTotalLambda

    // 4. 施加冲量
    // body1:
    v1 -= (deltaLambda / m1) * axis
    ω1 -= deltaLambda * invI1_r1PlusU_cross_axis

    // body2:
    v2 += (deltaLambda / m2) * axis
    ω2 += deltaLambda * invI2_r2_cross_axis
```

**关键**: 钳制是对**累积冲量 totalLambda** 而非增量 deltaLambda 做的。
这是 Erin Catto 的 **Accumulated Impulse Clamping** 方法，
比直接钳制增量收敛性好得多。

## 三、SpringPart — 软约束

对应 Erin Catto GDC 2011: "Soft Constraints: Reinventing The Spring"。

### 硬约束
```
bias = baumgarte_bias  // 直接用 Baumgarte 位置修正
softness = 0
effectiveMass = 1/K    // 不变
```

### 软约束 (弹簧)
```
// 输入: stiffness k, damping c, constraint_value C, dt

// softness (gamma): 弱化约束的程度
softness = 1.0 / (dt * (c + dt * k))

// bias: 弹簧驱动力
bias = original_bias + dt * k * softness * C

// 更新有效质量
effectiveMass = 1.0 / (K + softness)
```

然后在 SolveVelocityConstraint 中:
```
// bias计算变为:
actual_bias = bias + softness * totalLambda
lambda = effectiveMass * (jv - actual_bias)
```

`softness * totalLambda` 这项是**隐式阻尼**——
即使 damping ratio=0 也有内建阻尼 (隐式欧拉的特性)。

### 频率/阻尼比 → 刚度/阻尼系数 转换

```
omega = 2 * PI * frequency           // 角频率
k = effectiveMass * omega²           // 刚度
c = 2 * effectiveMass * damping * omega  // 阻尼系数
```

## 四、Warm Starting (热启动)

每帧开始时，用上一帧的 λ 作为初始值:

```
WarmStart(body1, body2, axis, warmStartRatio):
    totalLambda *= warmStartRatio    // ratio = dt_new / dt_old
    deltaLambda = totalLambda        // 整个累积量作为初始冲量施加

    // 施加冲量 (与 SolveVelocityConstraint 中的Step4相同)
    v1 -= (deltaLambda / m1) * axis
    ω1 -= deltaLambda * invI1_r1PlusU_cross_axis
    v2 += (deltaLambda / m2) * axis
    ω2 += deltaLambda * invI2_r2_cross_axis
```

Warm Starting 让求解器从一个已经接近正确的状态开始迭代，
**显著减少所需迭代次数** (通常从 50+ 降到 10)。
# Jolt 刚体物理核心算法 — 接触约束与摩擦

对应 Jolt 源码: `Physics/Constraints/ContactConstraintManager.cpp`。

## 一、每个接触点生成 3 个约束

对于每个接触点，Jolt 创建:

| 约束 | 轴向 | λ 范围 | 作用 |
|------|------|--------|------|
| 法线约束 (Non-Penetration) | 碰撞法线 n | [0, +∞) | 阻止穿透 |
| 摩擦约束1 (Friction1) | 切线 t1 | [-μλ_n, +μλ_n] | 阻止滑动 |
| 摩擦约束2 (Friction2) | 切线 t2 | [-μλ_n, +μλ_n] | 阻止滑动 |

三者都是 `AxisConstraintPart` 的实例，只是轴向和钳制范围不同。

## 二、约束组装 (SetupVelocityConstraints)

```
对每个接触流形中的每个接触点:

    // 接触点位置 = 两体接触位置的中点
    p = 0.5 * (worldPos1 + worldPos2)
    r1 = p - body1.centerOfMass    // body1质心到接触点
    r2 = p - body2.centerOfMass    // body2质心到接触点

    // ---- 计算相对速度 ----
    v_rel = body2.GetPointVelocityCOM(r2) - body1.GetPointVelocityCOM(r1)
    // GetPointVelocityCOM(r) = linearVel + angularVel × r

    normal_velocity = v_rel · normal

    // ---- 穿透深度 ----
    penetration = (worldPos1 - worldPos2) · normal
    // > 0 表示穿透，< 0 表示分离

    // ---- 计算法线约束的 bias ----
    // 1) 投机接触 (Speculative Contact):
    //    对于未穿透的点，bias阻止它们在本帧穿透
    speculative_bias = max(0, -penetration / dt)

    // 2) 恢复 (Restitution):
    if restitution > 0 且 normal_velocity < -minVelocityForRestitution:
        if normal_velocity < -speculative_bias:
            // 确实会碰上，施加恢复力
            // 减去 gravity*dt 以补偿帧初过度施加的重力
            bias = restitution * (normal_velocity - gravity_dt_dot_normal)
        else:
            bias = speculative_bias
    else:
        bias = speculative_bias

    // ---- 组装法线约束 ----
    nonPenetrationConstraint.CalculateConstraintProperties(
        body1, r1, body2, r2, normal, bias)

    // ---- 组装摩擦约束 ----
    if combinedFriction > 0:
        frictionConstraint1.CalculateConstraintProperties(
            body1, r1, body2, r2, tangent1, surfaceVelocity1)
        frictionConstraint2.CalculateConstraintProperties(
            body1, r1, body2, r2, tangent2, surfaceVelocity2)
    else:
        frictionConstraint1.Deactivate()
        frictionConstraint2.Deactivate()
```

### 恢复系数的关键细节

```
// 恢复力公式中减去重力分量:
bias = e * (v_rel_normal - gravity * dt · normal)

// 原因: 速度积分已在帧开头施加了重力，但碰撞在帧初已存在
// 所以本帧加的重力是"多余的"，需要从恢复计算中扣除
// 否则: 静止在地面的物体会因重力加速度被误认为需要弹跳
```

### 最小恢复速度阈值

```
// PhysicsSettings.h: mMinVelocityForRestitution = 1.0 m/s
// 只有 |v_normal| > 1 m/s 才施加恢复
// 这让物体更快安定 (避免无限小弹跳)
```

## 三、切线方向的构造

```
// 从碰撞法线 n 构建正交基 (n, t1, t2)
// Jolt 在 ContactConstraint 中存储:
//   mWorldSpaceNormal (压缩为半精度)
//   通过 GetTangents() 还原 t1, t2
//
// 常用方法:
if |n.x| < 0.57:  // ~1/sqrt(3)
    t1 = normalize(n × (1,0,0))
else:
    t1 = normalize(n × (0,1,0))
t2 = n × t1
```

## 四、速度约束求解 (每次迭代)

```
sSolveVelocityConstraint(constraint, mp1, mp2):
    t1, t2 = constraint.GetTangents()

    // ==== 先解摩擦 (因为非穿透更重要，放后面) ====
    for each contactPoint:
        if friction_active:
            // 计算摩擦冲量
            λ1 = frictionConstraint1.GetTotalLambda()  // 未钳制的值
            λ2 = frictionConstraint2.GetTotalLambda()

            // Coulomb 摩擦锥: |f_t| ≤ μ * f_n
            // 但用两轴分量的圆形截断 (比正方形更物理):
            total_sq = λ1² + λ2²
            max_λ = μ * nonPenetrationConstraint.totalLambda

            if total_sq > max_λ²:
                // 超出摩擦锥，等比例缩放到锥面上
                scale = max_λ / sqrt(total_sq)
                λ1 *= scale
                λ2 *= scale

            // 施加摩擦冲量
            frictionConstraint1.ApplyLambda(λ1)
            frictionConstraint2.ApplyLambda(λ2)

    // ==== 再解非穿透 ====
    for each contactPoint:
        // λ ∈ [0, +∞): 只能推不能拉
        nonPenetrationConstraint.SolveVelocityConstraint(
            body1, body2, normal, minLambda=0, maxLambda=FLT_MAX)
```

### 求解顺序的重要性

1. **先摩擦，后法线**: 法线约束放最后，因为在迭代求解器中，
   **最后求解的约束贡献最大**。非穿透比摩擦更重要。

2. **摩擦使用上一迭代的法线冲量**: `max_λ = μ * λ_normal_prev`。
   这就是为什么 `PhysicsSettings::mNumVelocitySteps ≥ 2` 才能让摩擦正常工作——
   第一次迭代时法线冲量为0 (或warm start值)，摩擦也是0。

3. **圆形摩擦锥**: `λ1² + λ2² ≤ (μ*λ_n)²`，比正方形 `|λ1|≤μλ_n, |λ2|≤μλ_n` 更物理正确，
   且各向同性 (不依赖切线方向的选取)。

## 五、材质合并

```
// 默认合并规则 (可自定义):
combinedFriction = sqrt(body1.friction * body2.friction)    // 几何平均
combinedRestitution = max(body1.restitution, body2.restitution)  // 取大
```
# Jolt 刚体物理核心算法 — 主循环与位置修正

## 一、PhysicsSystem::Update 全局流程

Jolt 的主循环在 `PhysicsSystem::Update(dt, collisionSteps, ...)` 中。
剥去线程调度和 Island 分组后，核心步骤如下:

```
PhysicsSystem::Update(dt):
    // ===== 阶段 0: 外部力 & 速度积分 =====
    for each active body:
        ApplyGravity(body, gravity, dt)
        body.motionProperties.ApplyForceTorqueAndDrag(dt)
        // 此时 body 的 linearVelocity/angularVelocity 已更新
        // 但 position/rotation 还没更新

    // ===== 阶段 1: 宽相 (Broadphase) =====
    FindCollidingPairs()   // AABB 交叉检测
    // 输出: 潜在碰撞对列表

    // ===== 阶段 2: 窄相 (Narrowphase) =====
    for each pair:
        GJK + EPA → 碰撞法线、穿透深度、接触点
        ClipPoly → 接触流形 (≤4点)
    // 输出: ContactConstraint 列表

    // ===== 阶段 3: Island 构建 =====
    // 把通过接触/关节相连的刚体分成独立的 island
    // 每个 island 独立求解 → 更好的并行 & 更快的收敛

    // ===== 阶段 4: 约束求解 =====
    for each island:
        // ---- Warm Start ----
        for each constraint:
            constraint.WarmStart(dt / prev_dt)

        // ---- 速度迭代 ----
        for iter = 0 to numVelocitySteps-1:  // 默认10次
            for each joint_constraint:
                joint.SolveVelocityConstraint(dt)
            for each contact_constraint:
                sSolveVelocityConstraint(contact, mp1, mp2)

    // ===== 阶段 5: 位置积分 =====
    for each active body:
        body.position += body.linearVelocity * dt
        body.rotation = AddRotationStep(body.rotation, body.angularVelocity * dt)
        // AddRotationStep: 轴角 → 四元数增量

    // ===== 阶段 6: 位置修正 (可选，默认2次) =====
    for iter = 0 to numPositionSteps-1:
        for each contact_constraint:
            SolvePositionConstraint(contact)
        for each joint_constraint:
            joint.SolvePositionConstraint(dt)

    // ===== 阶段 7: 睡眠检测 =====
    for each island:
        CheckSleepStatus(island)
```

## 二、关键: 半隐式 (Symplectic) 欧拉 vs 显式欧拉

```
// 显式欧拉 (不稳定):
x += v * dt
v += a * dt

// 半隐式欧拉 (Jolt 使用):
v += a * dt     // 先更新速度 (阶段0)
x += v * dt     // 用新速度更新位置 (阶段5)
```

两者只差顺序，但半隐式欧拉保辛 (symplectic)，
能量误差不会单调积累，长时间仿真也稳定。

## 三、位置修正 — Baumgarte 稳定化

### 问题

速度约束只保证"不再穿透更深"，但已存在的穿透不会消失。
需要额外施加位置修正来推开重叠物体。

### Baumgarte 方法

```
// PhysicsSettings.h:
//   mBaumgarte = 0.2 (法线方向穿透修正比例)
//   mLinearCastThreshold = 0.75 (穿透过深时改用线性投射)

position_error = penetration_depth  // > 0 表示穿透

if position_error > 0:
    // 修正量 = β * 穿透量
    correction = β * position_error

    // 按有效质量分配
    delta_x1 = -correction * normal * (m2 / (m1 + m2))
    delta_x2 = +correction * normal * (m1 / (m1 + m2))

    // 实际代码中更精确: 考虑旋转惯量
    // 通过类似 AxisConstraintPart 的 K 来分配
```

### 位置修正的 clamping

```
// 不设上限修正量，只修正:
//   max(0, penetration - slop)
// 即留一点 "slop" (mPenetrationSlop, 默认极小) 允许的穿透量
// 避免 jitter (物体在穿透边界反复弹跳)
```

### 位置迭代次数

```
PhysicsSettings:
    mNumPositionSteps = 2   // 默认2次位置迭代
    // 通常2次已足够把穿透推到视觉可接受范围
    // 增加到4次可在高质量堆叠中获得更好的稳定性
```

## 四、睡眠 (Sleep / Deactivation)

### 判定条件

```
// 当一个 island 中所有刚体满足:
//   |linearVelocity| < mPointVelocitySleepThreshold (默认极小)
//   |angularVelocity| < mPointVelocitySleepThreshold

// 且持续时间 > mTimeBeforeSleep (默认 0.5 秒)
// → 整个 island 进入睡眠
```

### 睡眠行为

```
// 睡眠的物体:
// 1) 不参与速度积分 (不施加重力)
// 2) 不参与约束求解
// 3) AABB 不更新 (但保留在宽相中)
// 4) 当新物体与之发生碰撞时被唤醒

// 这极大节省性能: 场景中通常 >80% 物体在睡眠
```

## 七、需注意的细节

### 重力不在约束前施加的"陷阱"

Jolt 在帧开头先把重力加到速度上。这意味着:
- 静止在地面上的物体帧初会有向下的速度 `g*dt`
- 然后约束求解器会产生法线冲量消除这个速度
- 因此恢复公式中要减去 `g*dt·n` (见 07 文件)

### 位置积分在约束求解之后

关键设计: 位置在约束求解器修正完速度后才积分。
这保证物体移动方向是"经过约束修正的"，
而不是"先移动到错误位置再推回来"。
# Jolt 刚体物理核心算法 — 仿写指南与简化策略

## 一、目标回顾

从 Jolt 的 ~10万行 C++ 工业级物理引擎中，提取核心算法，
写成**单文件、≤2000行**的简单刚体物理引擎。
支持: 凸体碰撞、摩擦、恢复、堆叠。

## 二、保留什么 / 砍掉什么

### ✅ 必须保留 (核心算法)

| 模块 | 理由 | 简化方式 |
|------|------|----------|
| 半隐式欧拉积分 | 稳定性基础 | 直接用，几行代码 |
| GJK | 凸体碰撞检测核心 | 保留完整算法，去掉SIMD |
| EPA | 穿透深度 | 保留但限制最大迭代(32) |
| 接触流形 (Clip) | 多点接触 = 稳定堆叠 | Sutherland-Hodgman，保留 |
| Sequential Impulse | 约束求解核心 | AxisConstraintPart 逻辑 |
| 摩擦 (Coulomb) | 物理真实感 | 圆形摩擦锥 |
| Warm Starting | 收敛速度 ×5 | 必须保留 |
| Baumgarte 位置修正 | 修穿透 | β=0.2, 简单几行 |

### ❌ 可以砍掉 (工业包装)

| 模块 | 理由 |
|------|------|
| 多线程 / Job 系统 | 单线程足够教学 |
| Island 分组 | 所有物体放一个 island |
| Broadphase (BVH) | 用 O(n²) 简单遍历 |
| 形状层次结构 | 只支持 Box + Sphere |
| 内存池 / 分配器 | 直接 new/delete 或栈 |
| Contact Cache / Manifold Cache | 每帧重新检测 |
| CCD (连续碰撞) | 用离散就行 |
| 角色控制器 | 与算法无关 |
| Soft body / Vehicle | 超出范围 |
| 压缩存储 (Half-float等) | 直接用 float |

### ⚠️ 可选 (看目标)

| 模块 | 建议 |
|------|------|
| 睡眠 | V1 不要，V2 加上 |
| 陀螺力矩 | V1 跳过，旋转物体少时影响小 |
| 关节约束 | V1 只做接触，V2 加铰链/球窝 |
| 投机接触 | V1 不要，高速场景才需要 |
| Soft constraint (Spring) | 帧率稳定时可不要 |

Vec3 SupportSphere(Vec3 dir, float radius) {

Vec3 SupportWorld(Body& body, Vec3 worldDir) {
    Vec3 localDir = body.rotation.Inverse() * worldDir;
    Vec3 localPt = SupportLocal(body, localDir);  // Box/Sphere
    return body.rotation * localPt + body.position;
}

// Minkowski 差 A-B 的 Support
Vec3 SupportMinkowski(Body& A, Body& B, Vec3 dir) {
    return SupportWorld(A, dir) - SupportWorld(B, -dir);
}

## 七、约束求解器模板

```cpp
// 通用 1-DOF 约束求解 (法线/摩擦都用)
float SolveConstraint1D(
    RigidBody& b1, RigidBody& b2,
    Vec3 r1, Vec3 r2,      // 质心到接触点
    Vec3 axis,              // 约束轴
    float bias,             // 目标速度偏移
    float& totalLambda,     // 累积冲量 (in/out)
    float minLambda, float maxLambda,
    float effectiveMass     // 预计算的 K⁻¹
) {
    // 计算约束速度
    Vec3 v1 = b1.linearVelocity + cross(b1.angularVelocity, r1);
    Vec3 v2 = b2.linearVelocity + cross(b2.angularVelocity, r2);
    float jv = dot(v2 - v1, axis);

    // 计算冲量增量
    float deltaLambda = effectiveMass * (jv - bias);

    // 累积冲量钳制
    float oldLambda = totalLambda;
    totalLambda = clamp(totalLambda + deltaLambda, minLambda, maxLambda);
    deltaLambda = totalLambda - oldLambda;

    // 施加冲量
    Vec3 impulse = axis * deltaLambda;
    b1.linearVelocity  -= impulse * b1.invMass;
    b1.angularVelocity -= b1.GetWorldInvInertia() * cross(r1, impulse);
    b2.linearVelocity  += impulse * b2.invMass;
    b2.angularVelocity += b2.GetWorldInvInertia() * cross(r2, impulse);

    return deltaLambda;
}

// 有效质量预计算
float ComputeEffectiveMass(
    RigidBody& b1, RigidBody& b2,
    Vec3 r1, Vec3 r2, Vec3 axis
) {
    // K = m1⁻¹ + m2⁻¹ + (I1⁻¹(r1×n))×r1 · n + (I2⁻¹(r2×n))×r2 · n
    float k = b1.invMass + b2.invMass;
    Vec3 t1 = cross(b1.GetWorldInvInertia() * cross(r1, axis), r1);
    Vec3 t2 = cross(b2.GetWorldInvInertia() * cross(r2, axis), r2);
    k += dot(t1 + t2, axis);
    return 1.0f / k;
}
```

## 二、核心数据结构定义

### 2.1 刚体 (Body)

从 Jolt 的 Body.h 和 MotionProperties.h 中提取的**最小必要字段**：

```
RigidBody {
    // === 几何状态 ===
    Vec3   position;          // 质心位置 (世界空间)
    Quat   rotation;          // 朝向四元数
    Shape* shape;             // 碰撞形状引用

    // === 运动状态 ===
    Vec3   linearVelocity;    // 线速度 (m/s)
    Vec3   angularVelocity;   // 角速度 (rad/s)

    // === 质量属性 (仅dynamic) ===
    float  invMass;           // 逆质量 (1/kg)
    Vec3   invInertiaDiag;    // 逆惯性张量对角线 D
    Quat   inertiaRotation;   // 惯性主轴旋转 R
    // 本体逆惯性 = R * diag(D) * R^-1
    // 世界逆惯性 = bodyRot * R * diag(D) * (bodyRot * R)^-1

    // === 力/扭矩累积 (每帧清零) ===
    Vec3   force;             // 累积力 (N)
    Vec3   torque;            // 累积扭矩 (N·m)

    // === 材质 ===
    float  friction;          // 摩擦系数 (无量纲, 通常0~1)
    float  restitution;       // 恢复系数 (0=完全非弹性, 1=完全弹性)

    // === 阻尼 ===
    float  linearDamping;     // 线性阻尼
    float  angularDamping;    // 角阻尼
    float  gravityFactor;     // 重力缩放 (1.0=正常)

    // === 类型 ===
    enum   motionType;        // Static / Kinematic / Dynamic
}
```

### 2.2 惯性张量的对角化存储

Jolt **不存储完整 3×3 惯性矩阵**。创建时通过 `MassProperties::DecomposePrincipalMomentsOfInertia()` 做特征分解：

$$I_{body}^{-1} = R \cdot \text{diag}(d_x,\ d_y,\ d_z) \cdot R^{-1}$$

- `invInertiaDiag` = $(d_x, d_y, d_z)$（3个float）
- `inertiaRotation` = $R$（1个Quat）

**世界空间逆惯性乘向量**的实际计算（MotionProperties.inl `MultiplyWorldSpaceInverseInertiaByVector`）：

```
result = bodyRotation * inertiaRotation 
       * (invInertiaDiag ⊙ (bodyRotation * inertiaRotation)^-1 * v)
```

即：先转到惯性主轴空间 → 逐分量乘对角线 → 转回世界空间。

## 三、完整物理更新流程 (每帧一步)

对应 PhysicsSystem.cpp 的 Job 依赖图，简化为单线程顺序：

### Step 0: 前处理
```
清零 force/torque 累积
调用用户 StepListener
```

### Step 1: 速度积分 — 力→速度
对每个 **Dynamic** body（MotionProperties.inl `ApplyForceTorqueAndDragInternal`）：

```
// 重力 + 外力 → 线速度
v += dt * (gravityFactor * gravity + invMass * force)

// 外扭矩 → 角速度 (需转到世界空间)
ω += dt * I_world_inv * torque

// 线性阻尼: 指数衰减的一阶泰勒近似
v *= max(0, 1 - linearDamping * dt)
ω *= max(0, 1 - angularDamping * dt)

// 速度钳制
if |v|² > maxLinVel² : v *= maxLinVel / |v|
if |ω|² > maxAngVel² : ω *= maxAngVel / |ω|
```

### Step 1.5: 陀螺力 (可选)
对标记了 `ApplyGyroscopicForce` 的body（MotionProperties.inl `ApplyGyroscopicForceInternal`）：

```
// 陀螺扭矩: T = -ω × (I·ω)  (Dzhanibekov 效应)
ω_local = (bodyRot * inertiaRot)^-1 * ω
I_local = 1/invInertiaDiag  (逐分量取倒)
L = I_local ⊙ ω_local       // 本体角动量
L_new = L - dt * (ω_local × L)

// 关键：归一化角动量长度以避免能量注入
L_new *= |L| / |L_new|

// 转回世界角速度
ω = (bodyRot * inertiaRot) * (invInertiaDiag ⊙ L_new)
```

### Step 2: 宽相位碰撞检测

// Jolt用QuadTree加速, 简化版直接O(n²):
for each pair (A, B):
    if AABB(A) ∩ AABB(B) ≠ ∅:
        narrowPhasePairs.add(A, B)

### Step 3: 窄相位碰撞检测 (GJK + EPA)

    if v·w < 0 且 v·w之平方 > |v|
